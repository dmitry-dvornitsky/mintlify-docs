## Цель

Дать каждому пользователю персональную точку входа в приложение — место где за одним взглядом видно всё важное.

## Что должно быть на дашборде

- **Мои проекты** — проекты, в которых участвует пользователь
- **Мой KPI** — прогресс по ключевым метрикам для конкретной роли
- Другие важные данные, актуальные при каждом открытии приложения

## Почему важно

Сейчас пользователь попадает на общий список Projects без персонализации. Dashboard даст немедленный контекст и снизит время до "начала работы".

Разные роли видят разный KPI:
- **SDR** — отправлено писем, назначено встреч, конверсия
- **Account Strategist** — статус по клиентам, дедлайны
- **Lead Researcher** — объём переданных лидов
- и т.д.
