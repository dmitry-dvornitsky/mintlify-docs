Client Portal — то, что видит клиент Belkins. Заменяет Client View из v1: не только дашборд, но и рабочие действия клиента — подтверждение встреч, аппрувы материалов, чат с командой. Цель: убрать переписку «в почте по кругу» и дать клиенту ощущение полного контроля.

## Разделы

### Dashboard

Упрощённая версия дашборда проекта: KPI-прогресс, воронка, каналы, последние positive-ответы (без внутренних инструментов, стоимостей и Gates).

### Meetings

Список встреч клиента со статусами. Действия клиента:

| Действие | Эффект |
|---|---|
| **Confirm** | Подтверждает участие — команда видит подтверждение |
| **Reschedule** | Выбор нового слота из календаря — пересогласование с лидом делает команда/агент |
| **Rate meeting** | После встречи: оценка качества лида (ICP fit) — попадает в [QA](/v2/admin/quality) |
| **Dispute status** | Клиент не согласен со статусом (например, No Show) — расхождение подсвечивается команде |

### Approvals

Очередь материалов, требующих согласия клиента:

- **Sequences** — тексты последовательностей перед запуском (если проект настроен на клиентский аппрув)
- **Target lists** — список компаний сегмента
- **Insights-страницы** — например, сформулированные ценностные предложения

Approve / Request changes (с комментарием). Просроченные аппрувы эскалируются AD.

### Reports

Архив еженедельных отчётов. Отчёт генерирует AI, проверяет AS, публикует платформа. Клиент может задать вопрос к отчёту — уходит в Chat.

### Chat

Канал коммуникации команда ↔ клиент, привязанный к проекту. Сообщения клиента попадают в My Day команды и (опционально) в Slack. История сохраняется в проекте — контекст не теряется при смене AD.

## Доступы

| Роль | Права |
|---|---|
| **Client Admin** | Всё выше + управление пользователями своей стороны |
| **Client Viewer** | Dashboard и Meetings без действий |

Приглашение клиента — из [Settings → Team](/v2/workspace/settings/team), секция «Клиентский доступ».

## Что клиент не видит

Внутренние инструменты (Inbox, Leads, стоимости, Usage, работа агентов), другие проекты, персональные данные команды за пределами имён. Показ AI-сводок клиенту всегда идёт через проверку человеком.

## MCP

`portal.publish_report`, `portal.request_approval`, `portal.get_approvals`, `portal.send_message` — команда и агенты взаимодействуют с порталом программно (например, SDR Agent запрашивает аппрув сиквенса у клиента).
