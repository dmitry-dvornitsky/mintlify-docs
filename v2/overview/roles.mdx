Права в v2 едины для всех поверхностей: роль определяет набор доступных действий в Action Layer, а UI, MCP и workflows лишь используют их. **AI-агент — такой же субъект прав, как человек**, плюс уровень автономии.

## Роли команды

| Роль | Зона ответственности | Основные модули |
|---|---|---|
| **Account Director (AD)** | Отношения с клиентом, апсейл, удержание | Projects, Admin → Clients, Client Portal |
| **Account Strategist (AS)** | Стратегия проекта, координация, качество | Strategy Hub, Insights, Gates |
| **SDR** | Переписка, назначение встреч | Inbox, Campaigns, Meetings |
| **Lead Researcher (LR)** | Качество лидов, контроль пайплайнов | Strategy Hub → Pipeline, Leads, Gates |
| **EDS** | Deliverability, здоровье ящиков | Settings → Channels, Inbox → Bounces |
| **Content Writer (CW)** | Тексты и месседжинг | Insights, Campaigns → Sequence review |
| **Sales** | Продажи (read-only по своим клиентам) | Projects (просмотр), Admin → Clients (свои) |

## Управленческие роли (Admin-модуль)

| Роль | Доступ |
|---|---|
| **C-level (CEO, COO, CFO)** | Весь Admin: Command Center, Clients, Team, Finance, Quality, System |
| **Operations Manager** | Admin без Finance-настроек: Clients, Team, Quality, Automation Monitor |
| **Finance Manager** | Admin → Finance: инвойсы, каталог сервисов, расходы |
| **Team Lead** | Admin → Team по своей функции (SDR Lead видит SDR-метрики) |

## AI-агенты

Каждому агенту назначаются роль-эквивалент (какие действия доступны) и **уровень автономии**:

| Уровень | Поведение |
|---|---|
| **L1 — Suggest** | Агент только предлагает: черновики, рекомендации. Ничего не выполняет |
| **L2 — Act with approval** | Выполняет действия, каждое проходит через Gate |
| **L3 — Autonomous in limits** | Действует сам в рамках лимитов; Gate только для исключений |
| **L4 — Fully autonomous** | Без гейтов (только для безопасных действий: чтение, отчёты) |

Полный список агентов и их настройки — в [Automation → AI Agents](/v2/automation/agents).

## Клиентские роли

| Роль | Возможности в Client Portal |
|---|---|
| **Client Admin** | Дашборд, встречи (подтверждение/перенос), аппрувы материалов, чат, управление пользователями своей стороны |
| **Client Viewer** | Только дашборд и список встреч |

Клиентские роли не имеют доступа к Workspace — только к [Client Portal](/v2/workspace/client-portal).
