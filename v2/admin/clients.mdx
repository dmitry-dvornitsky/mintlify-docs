Портфель всех клиентов Belkins для менеджмента. Отвечает на вопросы: у кого проблемы, кто продлится, где деньги под риском, кого можно расширить.

## Таблица

| Колонка | Описание |
|---|---|
| Client | Название + AD/AS проекта |
| **Health** | Health Score 0–100 (см. ниже) |
| Stage | Onboarding / Research / Outreach / Delivery / Renewal / Churned |
| KPI pace | on track / behind / ahead — из KPI-прогноза проекта |
| MRR | Текущая выручка клиента |
| Renewal | Дата конца контракта; подсветка < 90 дней |
| Team | Аватары команды проекта |

**Quick Filters:** Stage, At risk, Renewal ≤ 90d, By AD, By industry.

## Health Score

Композитный скор 0–100, считается непрерывно. Формула прозрачна — карточка клиента показывает разложение:

| Фактор | Вес | Сигналы |
|---|---|---|
| **Delivery** | 40% | KPI-выполнение за 3 месяца, тренд reply rate, качество встреч (QA) |
| **Engagement** | 25% | Активность клиента в Portal: логины, подтверждение встреч, скорость аппрувов, тон сообщений |
| **Financial** | 20% | Платёжная дисциплина, overdue-инвойсы |
| **Relationship** | 15% | Возраст контракта, эскалации, NPS-ответы, статус пилота |

Пороги: **● ≥70 healthy · ◐ 40–69 watch · ⚠ ниже 40 at risk**. Падение через порог → алерт AD и Operations + событие `client.health_dropped` (можно навесить workflow).

## Карточка клиента

Полный профиль для управленческих решений:

| Секция | Содержимое |
|---|---|
| Health breakdown | Из чего сложился скор, динамика по месяцам |
| Delivery | KPI-история, встречи, воронка, активные стратегии/кампании |
| Financial | Подписки, инвойсы, LTV, маржа клиента |
| Timeline | Ключевые события: контракт, эскалации, save-планы, апсейлы |
| **AI Save Brief** *(при риске)* | Диагноз причин + рекомендуемые действия; кнопка Create save plan создаёт план с задачами AD/AS |
| **AI Expansion Brief** *(при перевыполнении)* | Сигналы апсейла: KPI стабильно >110%, клиент просит новые сегменты — аргументы для расширения контракта |

## Renewal-процесс

За 60 дней до конца контракта workflow автоматически:
1. Создаёт Renewal prep задачу AD
2. Генерирует AI-сводку: результаты за контракт, ROI-аргументы, риски
3. Переводит клиента в Stage: Renewal — виден в Command Center

Итог фиксируется: Renewed (новый контракт в Billing) / Churned (с причиной — churn-аналитика строится из этих причин).
