Финансовый контур компании: биллинг всех клиентов, справочник сервисов, внешние расходы и маржа. Права: Finance Manager, C-level; проектный срез виден AD в [Settings → Client → Billing](/v2/workspace/settings/client#billing).

## Overview

North Star финансов: MRR/ARR по подпискам, собрано за месяц, дебиторка (Outstanding, из неё Overdue), валовая маржа. Тренды к прошлым периодам. Прогноз cash-in на месяц из графика инвойсов.

## Invoices

Все инвойсы всех клиентов (та же модель, что в проектном Billing, — здесь агрегат):

- Статусы: Draft / Created / Sent / Paid / Overdue; Source: Subscription / Synced (Wave)
- **Bulk Send to Wave**, экспорт
- **Overdue-автоматика:** напоминание клиенту (AI-черновик, утверждает Finance), алерт AD, влияние на Health Score. Эскалация по дням просрочки настраивается workflow'ом

## Service Catalog

Справочник сервисов Belkins — источник для подписок всех проектов:

| Поле | Описание |
|---|---|
| Service name | Например, «Lead Generation — Standard» |
| Default price / cycle | Подставляется в проект при добавлении, редактируется локально |
| Type | Recurring / One-time |
| Status | Active / Archived |

Изменение цены в каталоге не трогает существующие подписки — только новые.

## Costs

Внешние расходы по всем проектам (агрегат проектных Costs):

| Срез | Вопрос |
|---|---|
| By tool | Сколько уходит на OpenAI / Generect / Apollo / … — где давить на цену |
| By client | Во сколько обходится доставка каждого клиента |
| **Margin by client** | Выручка − расходы по каждому клиенту. Маржа ниже порога — алерт: клиент убыточен или near |
| By agent | Стоимость работы AI-агентов (LLM-расходы) — окупаемость автоматизации |

## Финансовые workflow по умолчанию

| Триггер | Действие |
|---|---|
| Наступила дата инвойса Recurring-подписки | Инвойс создан → Gate: Finance approve → Send to Wave |
| Инвойс Overdue 3 дня | AI-напоминание клиенту (approve Finance) + алерт AD |
| Маржа клиента < 40% два месяца | Задача Operations: разбор расходов |
| Контракт заканчивается, продления нет | Стоп будущих инвойсов + задача AD |
