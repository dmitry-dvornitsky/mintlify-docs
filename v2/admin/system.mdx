System — системный уровень платформы: кто и что может делать, какие агенты существуют, что подключено и полный след всех действий. Права: Operations, C-level.

## Permissions

Управление ролями (людей и клиентов): создание кастомных ролей, назначение действий из реестра, project-overrides. Матрица «роль × действие» — UI над Action Registry.

## Action Registry

Реестр всех действий платформы — **источник истины** для UI, MCP и workflows (см. [Архитектура → Action Layer](/v2/overview/platform-architecture#action-layer)). Для каждого действия: scope, roles, gate-политика, лимиты, audit-настройки, событие. Изменения версионируются и сами попадают в Audit Log.

## Agents

Глобальный реестр AI-агентов: включение/выключение, уровень автономии по умолчанию, роль-эквивалент, промпты (с версионированием и rollback), модель LLM, бюджеты. Подробное описание агентов — [Automation → Agents](/v2/automation/agents).

## Integrations

Организационные подключения: Wave, Unipile, Apollo, Generect, Hunter, LLM-провайдеры, webhook-эндпоинты. Статус, ключи, лимиты, расходы по каждой. Здесь же — выпуск **MCP-токенов** (scoped: набор действий × проекты × срок жизни) для внешних агентов.

## Automation Monitor

Живой контроль автоматики:

| Блок | Что показывает |
|---|---|
| **Live feed** | Что агенты делают прямо сейчас |
| **Pending gates** | Все ожидающие approve по компании — просроченные подсвечены |
| **Approve / undo rates** | По каждому агенту — деградация видна сразу |
| **AI spend** | Расходы LLM по агентам и проектам, против бюджетов |
| **Kill switch** | Пауза любого агента или всей автоматики — немедленно, одной кнопкой |

## Audit Log

Неизменяемый журнал всех действий: актор (👤 человек / ✦ агент / система), действие, объект, payload, результат, время. Фильтры: актор, действие, проект, период. Экспорт. Проектный срез доступен в Settings проекта.

<Note>Для действий агентов дополнительно хранится **reasoning trace** — на основании чего агент принял решение (какой контекст читал, какая уверенность). Это база для разбора инцидентов и улучшения промптов.</Note>
