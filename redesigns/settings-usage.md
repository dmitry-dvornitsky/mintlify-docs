# Redesign: Settings / Usage

## Проблемы

### 1. Название и позиционирование
"Usage" — слишком общее. Не ясно с первого взгляда что это про деньги и внешние инструменты, а не про то как команда использует платформу.

**Варианты переименования:**
- **"Costs"** — прямолинейно, сразу понятно
- **"Tool Costs"** — уточняет что речь про внешние инструменты
- **"Spending"** — более разговорный вариант

### 2. "Request Log" конфликтует с модулем Requests
Слово "Request" уже занято в системе (модуль Requests для поисков и импортов). Путаница.

**Варианты переименования:**
- **"Transaction Log"** — каждая строка = одна транзакция с внешним API
- **"API Call Log"** — точнее технически
- **"Activity Log"** — более нейтрально

### 3. Credits — несравнимая единица
OpenAI считает credits в токенах (31M за $57), Generect — в профилях (11K за $72). Сравнивать credits между инструментами бессмысленно — это вводит в заблуждение.

**Идея:** убрать колонку Credits из Breakdown, оставить только Cost и %. Credits оставить только в Request Log где они относятся к одному инструменту и имеют смысл.

### 4. 3 пустые карточки "Coming Soon"
Занимают место, выглядят незаконченно. Варианты что там могло бы быть:
- **Avg. daily spend** — средний расход в день за период
- **Projected monthly cost** — прогноз расхода до конца месяца на основе текущего темпа
- **Most expensive action** — топ фича по расходам

До готовности — убрать совсем или заменить одной карточкой с подпиской "Coming soon: spending forecasts".

### 5. Request Log слишком детальный
17,566 строк "Generate Campaign Sequence For Contact" одна за другой не дают инсайта. Нужен промежуточный уровень агрегации.

**Идея:** по умолчанию показывать агрегированный вид — Feature + день + количество вызовов + суммарная стоимость. Raw log доступен по клику или через Export.

### 6. Нет дефолтного периода
Не понятно за какой период показаны данные. Нужен дефолт (например, текущий месяц) и явная подпись периода рядом с Total Spent.

### 7. Module = "Other" непрозрачно
Непонятно что попадает в Other. Нужно либо расшифровать, либо убрать.
