## Проблема

Сейчас Strategy — это AI-сгенерированный текстовый документ. Он не связан с поиском лидов, не влияет на кампании, не упрощает работу SDR. Это знание на бумаге которое нужно переводить в действия вручную.

---

## Концепция: Strategy = Playbook

Strategy должна быть **конфигурационной сущностью**, а не документом. Один Researcher создаёт Playbook — вся команда запускает по нему поиск и кампании.

**Три слоя:**

```
Playbook
├── ICP (кого искать)       → автозаполняет Find Companies + Find Contacts
├── Messaging angle (почему это важно для них)  → AI пишет sequence
└── Sequence config (как строить outreach)      → преднастраивает Campaign
```

---

## Структура Strategy (Playbook)

### 1. Goal

Выбор типа кампании — определяет логику всего playbook:

| Goal | Описание |
|---|---|
| Extra Leads | Стандартный outreach — новые контакты |
| Re-engagement | Возврат к старым лидам |
| Event / Trigger | Реакция на событие (funding, новый CDO, переезд) |
| Referral | Тёплые рекомендации |

**Details** — свободное описание цели и контекста. AI использует это как "почему мы пишем именно сейчас".

---

### 2. ICP — кого искать

#### Компании
| Поле | Связь |
|---|---|
| Industry | → Find Companies / Industry |
| Exclude Industries | → Find Companies / Exclude Industries |
| Location | → Find Companies / Locations |
| Company size | → Find Companies / Company size |
| Type | → Find Companies / Type |
| Keywords | → Find Companies / Keywords |
| Revenue range | → Find Companies / Revenue range |
| Department headcount | → Find Companies / Department headcount |

#### Контакты
| Поле | Связь |
|---|---|
| Audience | → Find Contacts / Audience (First Priority) |
| Secondary Audience | → Find Contacts / Secondary Priority |
| Seniority / Level | → Find Contacts / Years in role |
| Location | → Find Contacts / Locations |
| Max contacts per company | → Find Contacts / Max per company |
| Recently posted on LinkedIn | → Find Contacts / Recently posted |

---

### 3. Messaging angle

Это то чего нет в текущем скетче — структурированный контекст для AI:

| Поле | Описание |
|---|---|
| **Core need** | Какую бизнес-проблему закрывает продукт для этого сегмента |
| **Value proposition** | Почему именно сейчас и именно мы |
| **Key talking points** | 3–5 тезисов которые AI должен использовать |
| **Avoid** | Что НЕ говорить (конкуренты, болезненные темы, клише) |
| **Social proof** | Какой кейс / результат упоминать |
| **Tone of voice** | Формальный / дружеский / провокационный / peer-to-peer |

---

### 4. Personalization hook

Какой кастомный сигнал AI использует для гиперперсонализации каждого письма:

| Вариант | Откуда берётся |
|---|---|
| Custom Column | Выбрать любую кастомную колонку из Settings → Columns — AI вставит значение в текст |
| LinkedIn activity | Последний пост контакта (если есть LinkedIn activity) |
| Job Signal | New in role / Recently joined — менять угол письма |
| Company news | Из Company AI Analysis |
| None | Без гиперперсонализации |

Это замена "Custom Column Message" из скетча, но более гибко — не просто выбор колонки, а выбор типа сигнала.

---

### 5. Sequence config

Преднастройка структуры кампании (необязательно):

| Поле | Описание |
|---|---|
| Channels | Email only / LinkedIn only / Email + LinkedIn |
| Steps | Количество шагов (например, 3 Email + 1 LinkedIn) |
| Delays | Интервалы между шагами (пример: 2 / 3 / 5 дней) |
| Mode | AI mode / Template mode |
| Tone of voice | (подтягивается из Messaging angle) |

## Потоки по ролям

### Researcher / Lead Research

**Задача:** найти компании и контакты под конкретную стратегию.

```
1. Leads → Find Companies
   → кнопка "Use strategy"
   → выбрать Playbook
   → все поля заполняются автоматически из ICP
   → проверить / скорректировать → запустить

2. Leads → Find Contacts
   → кнопка "Use strategy"
   → тот же Playbook
   → Audience, локации, параметры — всё заполнено
   → запустить
```

Researcher не вводит ничего вручную. Playbook = единый источник критериев.

---

### SDR

**Задача:** запустить кампанию.

```
Campaign → Create
  Step 1: выбрать Product + Strategy (Playbook)
  Step 2: выбрать контакты (уже найденные Researcher по этой стратегии)
  → Launch

AI пишет sequence автоматически:
  - берёт Product из Settings → Products (описание, конкуренты, возражения)
  - берёт Messaging angle из Strategy (need, value prop, talking points, avoid)
  - берёт Personalization hook (custom column / LinkedIn activity / job signal)
  - генерирует уникальное письмо для каждого контакта
```

SDR не трогает редактор последовательности если не хочет. Всё что нужно — Product + Strategy + контакты.

---

## Где живёт Strategy

**Не в Insights** — там знания о клиенте, а Strategy — это инструмент работы.

**Варианты:**
1. **Отдельный раздел в сайдбаре** — "Strategies" / "Playbooks" между Campaigns и Leads. Прямой доступ для всей команды.
2. **Внутри Settings** — рядом с Audiences и Criteria Groups (оба тоже про "кого искать"). Логично но сайдбар Settings растёт.
3. **Внутри Insights → Product** — остаётся где есть, но меняется структура. Минус: SDR ищет инструмент в базе знаний.

**Рекомендация:** вариант 1 — отдельный раздел. Strategy — это операционный инструмент, а не документ.

---

## Что меняется в существующих модулях

| Модуль | Изменение |
|---|---|
| **Find Companies** | Кнопка "Use strategy" → выбор Playbook → автозаполнение |
| **Find Contacts** | То же самое |
| **Campaign / Step 1** | Только Product + Strategy. Name генерируется автоматически по конвенции |
| **Campaign / Step 3** | Sequence pre-configured из Strategy. SDR может редактировать, но не обязан |
| **Insights / Strategy** | Текстовые страницы остаются как база знаний, но Playbook — отдельная сущность |
| **Settings / Audiences** | Audiences становятся частью Playbook ICP, не отдельным разделом |
| **Settings / Criteria Groups** | Criteria Groups тоже могут быть частью Playbook |

## Принятые решения

| Вопрос | Решение |
|---|---|
| Кто создаёт | Account Strategist — он задаёт ICP и messaging |
| Кто использует | Researcher выполняет поиск по ICP; SDR создаёт кампании |
| Где живёт | Отдельный раздел в сайдбаре проекта (наравне с Leads, Campaigns) |
| Привязка к продукту | 1 продукт → автоматически, без вопроса; 2+ продукта → выбор в начале создания |
