## В двух словах

Сейчас Strategy — это текстовый документ в Insights. Researcher и SDR читают его и вручную переводят в действия: сами заполняют критерии поиска, сами настраивают кампанию. Это медленно, неточно и зависит от того насколько внимательно человек прочитал документ.

**Новая концепция:** Strategy — это конфигурационная сущность. Account Strategist один раз задаёт ICP и messaging. Дальше Researcher нажимает одну кнопку и все поля поиска заполнены. SDR нажимает одну кнопку и AI пишет весь сиквенс. Вся команда работает строго в рамках одной стратегии.

---

## Роли и зоны ответственности

| Роль | Что делает | Когда |
|---|---|---|
| **Account Strategist** | Создаёт Strategy: задаёт ICP, messaging, personalization hook | Один раз при настройке нового сегмента |
| **Researcher** | Ищет компании и контакты выбирая стратегию — все критерии уже заполнены | При каждом поиске лидов |
| **SDR** | Создаёт кампанию выбирая стратегию — AI пишет сиквенс | При запуске нового outreach |

## Блок 1 — Создание стратегии (Account Strategist)

Стратегия создаётся один раз и используется многократно. Если в проекте **один продукт** — он привязывается автоматически. При **двух и более** — первый экран это выбор продукта.

### Шаг 1 / Goal — зачем пишем

Тип outreach определяет угол всех писем:

| Goal | Когда используется |
|---|---|
| **Extra Leads** | Стандартный outreach по новому сегменту |
| **Re-engagement** | Возврат к лидам которые не ответили ранее |
| **Event / Trigger** | Реакция на событие: смена роли, funding, найм, новость |
| **Referral** | Тёплые рекомендации через существующие связи |

**Details** — свободный текст: что именно происходит в этом сегменте сейчас, почему момент подходящий. AI использует это как "контекст запуска".

### Шаг 2 / Company ICP — каким компаниям пишем

Критерии компаний. Эти же поля появятся в Find Companies при выборе стратегии:

| Поле | Обязательно |
|---|---|
| Industry | ✓ |
| Exclude Industries | — |
| Location | ✓ |
| Company size (сотрудники) | ✓ |
| Type (public / private / non-profit) | — |
| Keywords | — |
| Exclude keywords | — |
| Revenue range (LinkedIn / другие источники) | — |
| Department headcount (размер отдела) | — |

### Шаг 3 / Contact ICP — каким людям пишем

Критерии контактов. Эти же поля появятся в Find Contacts при выборе стратегии:

| Поле | Обязательно |
|---|---|
| Audience (первичный набор должностей) | ✓ |
| Secondary Audience | — |
| Locations | — |
| Max contacts per company | — |
| Years in role (диапазон) | — |
| Recently posted on LinkedIn | — |

### Шаг 4 / Messaging — что говорим

Контекст для AI. Именно здесь разница между "хорошим письмом" и "письмом как пишет лучший SDR":

| Поле | Что даёт AI |
|---|---|
| **Core need** | Какую бизнес-проблему закрывает продукт для этого сегмента — AI выстраивает вокруг этого весь нарратив |
| **Value proposition** | Почему именно мы и именно сейчас — главный hook первого письма |
| **Key talking points** | 3–5 тезисов которые AI должен использовать — задаёт угол каждого шага |
| **Avoid** | Что НЕ говорить: клише, конкуренты, болезненные темы — AI не включит это никогда |
| **Social proof** | Кейс или результат для упоминания — AI встроит органично, не как рекламу |
| **Tone of voice** | Формальный / дружеский / провокационный / peer-to-peer — определяет стиль всего сиквенса |

### Шаг 5 / Personalization hook — как персонализируем каждое письмо

Один сигнал который AI использует чтобы письмо выглядело написанным конкретно этому человеку:

| Вариант | Откуда данные | Что даёт |
|---|---|---|
| **Custom Column** | Выбрать колонку из Settings → Columns | AI встраивает значение органично в текст |
| **LinkedIn activity** | Последний пост контакта (из Requests → LinkedIn Activity) | AI ссылается на конкретную публикацию |
| **Job Signal** | New in role / Recently joined | AI меняет угол: "поздравляю с новой ролью..." |
| **Company news** | Результат Company AI Analysis | AI ссылается на конкретное событие компании |
| **None** | — | Персонализация только через ICP-данные |

## Блок 2 — Поиск компаний (Researcher)

**До Strategy:** Researcher вручную заполняет все поля каждый раз — Industry, Location, Size, Keywords, Revenue. Риск ошибки, потеря времени, несоответствие стратегии.

**После Strategy:**

```
Leads → Find Companies
→ кнопка "Use strategy"
→ выбрать стратегию из списка
→ все поля Company ICP заполнены автоматически
→ Researcher проверяет, при необходимости корректирует
→ Запустить поиск
```

**Что меняется в Find Companies:**
- Появляется кнопка **"Use strategy"** рядом с заголовком формы
- После выбора стратегии все поля Company ICP подставляются
- Поля **редактируемые** — стратегия рекомендует, Researcher корректирует под конкретный запуск
- Найденные компании **тегируются** стратегией: `Strategy: CFO UK Construction`
- В таблице Companies появляется колонка **Strategy** — видно откуда пришла компания

---

## Блок 3 — Поиск контактов (Researcher)

Аналогично Find Companies — тот же принцип:

```
Leads → Find Contacts
→ кнопка "Use strategy"
→ выбрать ту же стратегию
→ Audience, локации, параметры — всё заполнено
→ Запустить поиск
```

**Также:** Find Contacts in Companies — при поиске контактов внутри уже найденных компаний стратегия подставляется автоматически если компании были найдены по этой стратегии.

**Что меняется в Find Contacts:**
- Кнопка **"Use strategy"** в форме
- После выбора: Audience → подставляется, Locations, Max per company, Years in role, LinkedIn filter — всё из Contact ICP
- Контакты тегируются: `Strategy: CFO UK Construction`
- В таблице Contacts колонка **Strategy** показывает принадлежность

## Блок 4 — Запуск кампании (SDR)

**До Strategy:** SDR проходит 8 шагов вручную — заполняет описание кампании, настраивает сиквенс, подбирает tone of voice, думает что писать.

**После Strategy:**

```
Campaigns → Create
→ Step 1: выбрать Strategy
  (Product подставляется автоматически если один,
   или выбирается вместе со стратегией)
→ Step 2: контакты
  — по умолчанию показываются контакты тегированные этой стратегией
  — SDR может добавить другие
→ AI строит сиквенс автоматически
→ Step N: Preview — посмотреть что написал AI
→ Launch
```

**Что меняется в Campaigns:**
- Step 1 упрощается: только **Strategy** (+ Product если несколько)
- Step 2 Contacts: фильтр по умолчанию = контакты этой стратегии
- Step 3 Sequence: AI генерирует сиквенс сразу, без пустого редактора. SDR видит готовые письма и может отредактировать — но не обязан
- Campaign Name автогенерируется по конвенции: `{Client} / {Goal} / {Segment} / {Date}`

## Блок 5 — Что строит AI и откуда берёт данные

При генерации каждого письма AI собирает контекст из трёх источников:

```
┌─────────────────────────────────────────────────────────┐
│                    AI CONTEXT                            │
│                                                         │
│  Product (из Insights)                                  │
│  ├── Описание продукта                                  │
│  ├── Конкуренты и отличия                               │
│  ├── Типичные возражения и ответы                       │
│  └── Pricing / Solution                                 │
│                                                         │
│  Strategy / Messaging                                   │
│  ├── Goal + Details (почему пишем сейчас)               │
│  ├── Core need (проблема этого сегмента)                │
│  ├── Value proposition (наш ответ)                      │
│  ├── Key talking points (что включить)                  │
│  ├── Avoid (что исключить)                              │
│  ├── Social proof (какой кейс)                          │
│  └── Tone of voice (стиль)                              │
│                                                         │
│  Contact / Company (данные конкретного лида)            │
│  ├── Personalization hook (custom column / LinkedIn /   │
│  │   job signal / company news)                         │
│  ├── Job title, company, location                       │
│  └── AI Qualification result (если запускался)          │
└─────────────────────────────────────────────────────────┘
```

Итог: каждое письмо уникально для конкретного контакта, но строго в рамках стратегии. AI не импровизирует — он следует messaging guidelines.

## Что меняется в каждом модуле

### Strategies (новый модуль)
- Таблица стратегий: Name, Product, Goal, Contacts found, Campaigns, Updated, Status
- Создание через 5-шаговый wizard
- Карточка стратегии: ICP summary + Messaging + список связанных кампаний и поисков

### Leads / Find Companies
- Кнопка **"Use strategy"** в форме поиска
- Найденные компании получают тег стратегии
- Колонка Strategy в таблице Companies

### Leads / Find Contacts
- Кнопка **"Use strategy"** в форме поиска
- Контакты получают тег стратегии
- Колонка Strategy в таблице Contacts
- Quick Filter: "By strategy" — показать только контакты конкретной стратегии

### Campaigns / Create
- Step 1: только Strategy (+ Product если несколько)
- Step 2: default filter = контакты этой стратегии
- Step 3: AI генерирует готовый сиквенс, не пустой редактор
- Name автогенерируется

### Settings / Audiences
- Audiences остаются — они используются внутри Contact ICP стратегии
- Не исчезают, но больше не нужно идти в Settings чтобы понять ICP — всё в стратегии

### Insights / Strategy (текстовые страницы)
- Остаются как база знаний (Needs Solutions And Roles и др.)
- Но больше не являются операционным инструментом — это справочник

## Принятые решения

| Вопрос | Решение |
|---|---|
| Кто создаёт | Account Strategist |
| Кто использует | Researcher (поиск) + SDR (кампании) |
| Где живёт | Отдельный раздел в навигации проекта |
| Привязка к продукту | 1 продукт → авто; 2+ → выбор при создании |
| Теги на лидах | Да — компании и контакты тегируются стратегией |
| Автогенерация имени кампании | Да — по конвенции из стратегии |

## Открытые вопросы

| Вопрос | Почему важно решить до разработки |
|---|---|
| Версионирование стратегии | Кампания снимает снапшот стратегии в момент создания и работает по нему. Изменения стратегии не влияют на активные кампании — в backlog |
| ~~Можно ли запустить кампанию без стратегии?~~ | ✅ **Нельзя** — Strategy обязательна при создании кампании |
| ~~Что видит Researcher в форме — locked поля или editable?~~ | ✅ **Editable** — стратегия рекомендует, Researcher может скорректировать под конкретный запуск |
| Auto-generate стратегии из Insights? | AI предлагает ICP и messaging на основе заполненных Product/Client страниц |

## План разработки

Порядок важен: Strategies должны существовать до того как Leads и Campaigns начнут их использовать. Внутри каждой фазы задачи независимы и могут идти параллельно.

---

### Фаза 1 — Новый модуль Strategies

Всё что нужно чтобы стратегия существовала как сущность. Без этого ничего дальше не работает.

| # | Задача |
|---|---|
| 1.1 | **DB: Strategy entity** — поля: name, product_id, goal, details, company_icp (JSON), contact_icp (JSON), messaging (JSON), personalization_hook (type + column_id), status, created_by, created_at |
| 1.2 | **DB: Campaign.strategy_snapshot** — при создании кампании сохранять копию стратегии (JSON) чтобы изменения не влияли на активные кампании |
| 1.3 | **UI: Таблица стратегий** — Name, Product, Goal, Contacts found, Campaigns, Updated, Status. Sort, Filter, View |
| 1.4 | **UI: Wizard создания** — 5 шагов: Goal → Company ICP → Contact ICP → Messaging → Personalization hook. Если 1 продукт — шаг выбора продукта скрыт |
| 1.5 | **UI: Страница стратегии** — просмотр всех полей, кнопка Edit, список связанных кампаний и поисков |

### Фаза 2 — Leads: тегирование и автозаполнение

Добавляем стратегию в поиск лидов. Результат: компании и контакты знают по какой стратегии они найдены.

| # | Задача |
|---|---|
| 2.1 | **DB: Company.strategy_id** — связь компании со стратегией по которой она найдена |
| 2.2 | **DB: Contact.strategy_id** — то же для контактов |
| 2.3 | **UI: Find Companies — "Use strategy"** — кнопка/селектор стратегии в форме. При выборе: все поля Company ICP заполняются из стратегии. Поля редактируемые |
| 2.4 | **UI: Find Contacts — "Use strategy"** — то же: Audience, Locations, Max per company, Years in role, LinkedIn filter |
| 2.5 | **UI: Таблица Companies** — добавить колонку Strategy |
| 2.6 | **UI: Таблица Contacts** — добавить колонку Strategy + Quick Filter "By strategy" |
| 2.7 | **Logic: Find Contacts in Companies** — если компании найдены по стратегии, автоматически предлагать ту же стратегию для поиска контактов |

### Фаза 3 — Campaigns: обязательная стратегия и упрощение wizard'а

Кампания теперь требует стратегию и использует её для генерации сиквенса.

| # | Задача |
|---|---|
| 3.1 | **UI: Campaign Step 1** — убрать Description и Name. Оставить: Strategy (required) + Product (если несколько). Имя кампании автогенерируется: `{Client} / {Goal} / {Segment} / {Date}` |
| 3.2 | **Logic: Strategy обязательна** — нельзя перейти к Step 2 без выбранной стратегии |
| 3.3 | **UI: Campaign Step 2 (Contacts)** — default filter = контакты тегированные выбранной стратегией. Можно добавить другие вручную |
| 3.4 | **Logic: Strategy snapshot** — при сохранении кампании записать campaign.strategy_snapshot из текущей версии стратегии |

---

### Фаза 4 — AI: контекстная генерация сиквенса

Самая сложная фаза — AI должен строить сиквенс из структурированных данных, а не свободного текста.

| # | Задача |
|---|---|
| 4.1 | **AI context pipeline** — собрать контекст при генерации: Product (из Insights) + Strategy.messaging + стратегии snapshot |
| 4.2 | **Personalization hook resolver** — для каждого контакта получить значение hook'а: custom column value / LinkedIn post / job signal date / company AI analysis result |
| 4.3 | **UI: Campaign Step 3** — не пустой редактор, а AI генерирует письма сразу при входе на шаг. SDR видит готовый сиквенс и редактирует если нужно |
| 4.4 | **AI prompt engineering** — промпты используют structured fields (talking_points[], avoid[], social_proof) а не свободный текст описания кампании |
