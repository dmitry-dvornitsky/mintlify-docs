## Проблемы текущего дизайна

### 1. Табы по ролям — неудобная навигация
6+ табов (AS, DL, SDR, EDS, LR, Clients, Pending) означает: чтобы увидеть всю команду проекта нужно кликать по каждому табу. Невозможно быстро понять кто вообще работает на проекте.

### 2. Клиент смешан со стаффом
"Clients" как таб рядом с ролями сотрудников — принципиально разный тип пользователя. У клиента другие права, другой интерфейс (Client View), другой флоу добавления. Их нельзя мешать в одну навигацию.

### 3. Два способа добавить — непонятная разница
"Invite member" и "+ Add member" рядом — неясно чем они отличаются.

### 4. Pending скрыт в отдельном табе
Ожидающие инвайты никак не бросаются в глаза — легко пропустить что кто-то не принял приглашение.

---

## Предложение

### Структура страницы

Две чёткие секции на одной странице:

```
Team
├── Сотрудники Belkins
└── Клиентский доступ
```

---

### Секция 1 — Сотрудники Belkins

Одна таблица со всеми сотрудниками проекта. Никаких табов по ролям.

**Колонки:** Avatar + Name, Role, Email, Status (Active / Pending)

**Filters / Quick Filters:** по роли — для тех кто хочет отфильтровать

**Actions:**
- **+ Add member** — попап: выбрать пользователя из системы + назначить роль
- **Invite** — отправить email-приглашение внешнему человеку

Pending-инвайты видны прямо в таблице (Status = Pending + дата отправки + кнопка Resend).

---

### Секция 2 — Клиентский доступ

Отдельный блок с другим визуальным весом. Показывает кто из клиентской стороны имеет доступ к проекту.

```
┌─────────────────────────────────────────────────┐
│ Клиентский доступ                    [+ Invite] │
│                                                  │
│ john@client.com     Client View    Active        │
│ sarah@client.com    Client View    Pending ↺     │
└─────────────────────────────────────────────────┘
```

**Client View** — ограниченный просмотр: клиент видит только дашборд и метрики, не видит внутренние инструменты команды.

Ссылка: "Что видит клиент →" ведёт на описание Client Cabinet.

---

### Итог

| Сейчас | После |
|---|---|
| 6+ табов | 1 страница, 2 секции |
| Клиент смешан со стаффом | Чётко разделены |
| Pending скрыт в табе | Виден в статусе строки |
| Два непонятных action | Add member / Invite с чёткими ролями |
