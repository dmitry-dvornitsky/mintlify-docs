## Проблема

Текущий путь: **Leads → Requests → Find Companies → {Сегмент} → New Search**

- 4 уровня навигации для базового действия
- "Request" — имплементационный термин, непонятный пользователю
- Пользователь вручную создаёт Search #1, #2, #3 для одного сегмента
- "New search" звучит как начало нового поиска, а не продолжение текущего
- LinkedIn-лимиты — скрытая деталь о которой пользователь не должен думать

## Предложение

### 1. Переименовать Request → Segment

Пользователь думает "выгружу сегмент Healthcare 51-200", а не "создам реквест". Segment отражает суть — именованный набор ICP-критериев.

### 2. Встроить Segments в Leads

Убрать Requests как отдельный модуль. Добавить таб Segments прямо в Leads:

```
Leads
├── Companies  — все компании
├── Contacts   — все контакты
└── Segments   — именованные ICP-сегменты
```

### 3. Новый флоу Find Companies

**Step 1** — Критерии (без изменений)

**Step 2** — Preview & Launch:
- Name *
- How many companies? (число)
- Apply DNC List (toggle, по умолчанию ON)
- Skip Already Searched (toggle, по умолчанию ON)
- Run AI Analysis after completion (toggle)

→ Кнопка **"Start"** — создаёт Segment и сразу запускает первый батч. Система сама разбивает на батчи под капотом.

### 4. Страница Segment

```
[← Back]  Requests / Find Companies / Healthcare 51-200    [Get more] [⋯]

┌─────────────────┐  ┌──────────────────────┐  ┌──────────────┐
│ 1574 found      │  │ ~826 remaining       │  │ $0.0064/co.  │
└─────────────────┘  └──────────────────────┘  └──────────────┘

Progress: ████████████░░░░░░░  65% of segment

[ Companies ]  [ History ▾ ]

[таблица компаний]
```

**"Get more"** — попап с одним полем: сколько компаний добавить. DNC и Skip включены по умолчанию, не показывать каждый раз.

**History** — свёрнута по умолчанию. Раскрывается для прозрачности. Показывает батчи (бывшие Search #1, #2, #3) со статусом и количеством.

### 5. Что делать с остальными табами Requests

Import Companies, Import Contacts, AI Analysis, LinkedIn Activity — это разовые операции, не сегменты. Вынести в отдельный лёгкий раздел **History** или **Operations** (например, в Settings или как скрытый служебный раздел).
