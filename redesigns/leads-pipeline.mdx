## Проблема

Сегодня Researcher выполняет последовательность ручных операций, каждая из которых требует возвращения в приложение:

```
Find Companies          → подождать → вернуться
Company AI Analysis     → подождать → вернуться
Find Contacts           → подождать → вернуться
Phone Enrichment        → подождать → вернуться
LinkedIn Activity       → подождать → вернуться
Column Enrichment       → подождать → вернуться
                                      ↓
                              Контакты готовы
```

Между каждым шагом — пустое ожидание. Researcher не добавляет ценность, он просто нажимает кнопки. Этот процесс не масштабируется и не позволяет SDR работать самостоятельно.

---

## Концепция: Research Pipeline

Вместо ручных шагов — **конфигурируемый пайплайн**. Account Strategist или Researcher настраивает шаги один раз, нажимает "Run" и уходит. Система выполняет все шаги автоматически, уведомляет о завершении.

```
Настройка (1 раз)          Выполнение (автоматически)
━━━━━━━━━━━━━━━━━━━        ━━━━━━━━━━━━━━━━━━━━━━━━━
Step 1: Find Companies  →  Поиск компаний по ICP
Step 2: AI Analysis     →  Фильтрация по критериям
Step 3: Find Contacts   →  Поиск контактов в квалиф. компаниях
Step 4: Enrichments     →  Обогащение данных параллельно
                                     ↓
                           N контактов готовы для кампании
```

Pipeline конфигурируется **внутри Strategy** — ICP уже есть, шаги пайплайна логично живут там же. Researcher или SDR просто выбирают стратегию, вводят желаемое количество компаний и нажимают Run.

---

## Шаги пайплайна

### Step 1 — Find Companies

**Источник данных:**
- **LinkedIn search** — поиск по ICP из Strategy (Company ICP автозаполнен, редактируемый)
- **Import CSV** — загрузить готовый список компаний
- **Import LinkedIn URLs** — вставить список URL

**Настройки запуска:**

| Параметр | Описание |
|---|---|
| Target count | Сколько компаний найти (например, 500) |
| Apply DNC | Пропустить компании из Do Not Contact (по умолчанию: вкл) |
| Skip existing | Не дублировать компании уже в системе (по умолчанию: вкл) |
| Small companies first | Начинать с маленьких — легче добраться до ЛПР |

**Выход:** N компаний, тегированных стратегией. Передаются в следующий шаг.

---

### Step 2 — Company AI Analysis *(опционально)*

AI обходит сайты найденных компаний и оценивает соответствие ICP.

**Настройки:**

| Параметр | Описание |
|---|---|
| Criteria Group | Набор критериев из Strategy (или выбрать вручную) |
| Pass only Qualified | Передать в следующий шаг только компании с результатом ✅ Qualified |

**Выход:** отфильтрованный список компаний. Если Pass only Qualified включён — в Step 3 идут только подходящие.

<Note>Этот шаг увеличивает время выполнения и стоимость. Рекомендуется для сегментов где много нерелевантных компаний по формальным критериям ICP.</Note>

---

### Step 3 — Find Contacts

Поиск контактов внутри компаний прошедших через предыдущие шаги.

**Настройки:**

| Параметр | Описание |
|---|---|
| Audience | Из Contact ICP стратегии (автозаполнен, редактируемый) |
| Secondary Audience | Запасная аудитория |
| Max per company | Максимум контактов с одной компании |
| Years in role | Диапазон лет в текущей должности |
| Recently posted on LinkedIn | Только активные профили |
| Apply Contacts DNC | Пропустить контакты из DNC (по умолчанию: вкл) |

**Выход:** N контактов, тегированных стратегией. Передаются в шаг обогащения.

---

### Step 4 — Enrichments *(опционально, несколько одновременно)*

Операции обогащения запускаются **параллельно** — не нужно ждать каждую по очереди.

| Enrichment | Откуда | Зачем |
|---|---|---|
| **Phone numbers** | Apollo | Телефоны для контактов |
| **LinkedIn Activity Score** | LinkedIn профили | Оценка "живости" аккаунта — кому писать в LinkedIn |
| **Custom columns** | AI (сайт / LinkedIn профиль) | Заполнить кастомные поля по инструкциям из Settings → Columns |
| **Email validation** | QuickEmailVerification и др. | Подтвердить валидность email до отправки |

**Выход:** контакты с обогащёнными данными, готовы к добавлению в кампанию.

---

## Гейты — точки ручного контроля *(опционально)*

Для случаев когда нужна человеческая проверка качества — между шагами можно поставить **Gate**.

```
Step 1: Find Companies
        ↓
    [GATE] ← Researcher получает уведомление
             Проверяет список, нажимает "Approve → Continue"
             или "Stop pipeline"
        ↓
Step 2: AI Analysis
        ↓
    [GATE] ← необязательно
        ↓
Step 3: Find Contacts
```

Без Gate — шаги переходят автоматически. Gate имеет смысл когда компаний ищется впервые для нового сегмента.

## Продолжение поиска — "Find more"

LinkedIn отдаёт ограниченное количество результатов за один запрос. Чтобы найти больше компаний под ту же стратегию — кнопка **"Find more companies"** на странице стратегии.

```
Strategy → Research Pipeline
→ [▶ Run Pipeline]  [+ Find more companies]
                            ↓
              Модальное окно:
              "Find [N] more companies"
                            ↓
              Step 1 запускается заново:
              те же ICP-параметры + Skip existing: ON
                            ↓
              Новые компании проходят через Step 2 → 3 → 4
                            ↓
              Результаты добавляются к существующему пулу
```

**Ключевые свойства:**
- Skip existing всегда включён — нет дублей
- Все последующие шаги (AI Analysis, Find Contacts, Enrichments) запускаются автоматически для новых компаний
- Можно нажимать "Find more" сколько угодно раз — каждый раз получаем новую порцию
- История каждого запуска фиксируется в Pipeline History

Это заменяет текущий механизм "Searches внутри Request" — более простая ментальная модель.

## Лимиты для SDR

Account Strategist задаёт лимиты в настройках пайплайна:

| Лимит | Описание |
|---|---|
| **Max companies per run** | Максимум компаний которые SDR может запросить за один запуск (например, 200) |
| **Max total companies** | Максимум компаний по этой стратегии вообще (например, 1000) |

SDR видит лимит в модальном окне запуска и не может его превысить. Если лимит достигнут — кнопка "Find more" недоступна, с объяснением.

AS и Researcher могут запускать без лимитов или с более высокими лимитами (по роли).

## Флоу по ролям

### Account Strategist — настраивает пайплайн

```
Strategies → Strategy → Research Pipeline
→ Настроить шаги (один раз при создании стратегии)
→ Сохранить
→ Готово — пайплайн привязан к стратегии навсегда
```

Researcher и SDR используют этот пайплайн нажимая "Run" без настройки.

---

### SDR — запускает самостоятельно

```
Strategies → выбрать Strategy → Research Pipeline
→ [▶ Run Pipeline]
→ Ввести Target count (сколько компаний искать)
→ Run
→ Получить уведомление: "Pipeline complete — 312 contacts ready"
→ Campaigns → Create → выбрать Strategy → контакты уже там
```

SDR не видит детали ICP, не трогает форму поиска. Один экран, два клика.

---

### Researcher — расширенный режим

Researcher может:
1. Запускать пайплайн как SDR
2. Смотреть прогресс каждого шага
3. Обрабатывать Gates (проверять и approve/reject)
4. Запускать отдельные шаги вручную (advanced mode)
5. Создавать новые пайплайны для нестандартных сегментов

Роль Researcher смещается: **от "нажималы кнопок" к "контролёру качества"**.

## Что меняется в каждом модуле

### Strategy (расширение)
- Новая вкладка **"Research Pipeline"** на странице стратегии
- Поля в Strategy: добавить Criteria Group (для Step 2) и Enrichments (для Step 4)
- Кнопка **"Run Pipeline"** — главный CTA на странице стратегии
- Статистика: "Last run: Jun 10 · 312 contacts found"

### Leads (упрощение)
- **Find Companies** и **Find Contacts in Companies** остаются как **Advanced mode**
- Primary flow — через Pipeline из Strategy, не через Leads напрямую
- Новый баннер прогресса активного пайплайна в таблицах Companies и Contacts
- Колонка **Strategy** в таблицах — откуда пришёл лид
- Quick Filter: по стратегии

### Requests (трансформация → Pipeline History)
- Переименовать в **History** или **Pipeline Runs**
- Основной контент: история запусков пайплайнов (дата, стратегия, шаги, результат, стоимость)
- Детальная страница каждого запуска: сколько компаний/контактов на каждом шаге, сколько отфильтровано AI Analysis, сколько прошло
- Ручные операции (из старого Requests) уходят в archived/advanced раздел

### Settings → Criteria Groups
- Criteria Groups привязываются к Strategy напрямую
- В Strategy → Research Pipeline → Step 2 выбирается конкретный Criteria Group

### Alert Center (новый)
- Pipeline events: завершение, Gate, ошибка
- Интеграция со Slack (уже есть в Settings → Slack)

## Что остаётся ручным

Не всё нужно автоматизировать. Остаётся ручным намеренно:

| Действие | Почему ручное |
|---|---|
| Import Companies (CSV/URLs) | Researcher получает список из внешнего источника — это ввод данных, не поиск |
| Проверка Gate | Качественная оценка требует человека |
| Запуск пайплайна вручную | Пайплайн не запускается по расписанию сам — это осознанное действие |
| Редактирование контактов | Данные конкретного контакта всегда требуют проверки |
