Продукт состоит из трёх поверхностей: **Workspace** (delivery-команда), **Admin** (менеджмент) и **Client Portal** (клиенты, отдельный вход и свой layout). Роль определяет доступные поверхности — [матрица](/v4/platform/roles).

Ключевое решение навигации: **у Workspace нет раздела Projects**. Delivery-человек живёт в [Home](/v4/workspace/home) — там его очередь дел и его проекты. Управление списком проектов (создание, паузы, архив) — в [Admin → Projects](/v4/admin/projects).

## Кто использует

| Роль | Что видит |
|---|---|
| SDR, LR, EDS, CW | Только Workspace |
| AS, AD | Workspace + переключатель в Admin (свои разделы) |
| Ops, C-level | Workspace + весь Admin |
| Клиент | Только Client Portal — этот layout его не касается |

## Как работает

### Хедер (всегда виден)

```
[🏠]  [Проект: Acme Corp ▾]      [🔍 ⌘K]  [✦ Ask AI]  [+]  [🔔]  [Admin ⇄]  [👤]
```

| Элемент | Поведение |
|---|---|
| **🏠 Home** | Первый элемент хедера, тёмный блок-якорь — занял место логотипа ([D-012](/v4/decisions): логотипа в рабочем хедере нет, бренд живёт на логине и в Portal). Клик → Home: глобальный экран пользователя — очередь дел и карточки его проектов. Home живёт в хедере, а не в сайдбаре ([D-011](/v4/decisions)) и рендерится **без сайдбара** — у глобального экрана нет проектного контекста |
| **Project switcher** | Текущий проект + дропдаун: поиск, мои проекты, недавние. Вне проекта — скрыт. Переключение сохраняет текущий модуль (был в Inbox → попадаешь в Inbox другого проекта) |
| **Глобальный поиск ⌘K** | Ищет везде: компании, контакты, кампании, треды, встречи, проекты, AI-документы и материалы (по распарсенному содержимому). Результаты сгруппированы по типу, показывают проект-источник. Недавние объекты — до ввода запроса |
| **✦ Ask AI** | Открывает правую панель AI-ассистента с контекстом текущего экрана: отвечает на вопросы по данным («почему у Acme упал reply rate?»), выполняет действия через Action Layer строго в правах пользователя и через те же [Gates](/v4/platform/automation). Панель живёт поверх экрана, диалог сохраняется при навигации |
| **Quick create [+]** | Контекстные действия по правам: задача, встреча, импорт лидов, кампания; для AS/AD — новый проект (ведёт в [визард](/v4/admin/projects/create)) |
| **Уведомления 🔔** | Счётчик непрочитанных, центр уведомлений: группировка по проекту, mark all read, переход к объекту. Настройки — [карта уведомлений](/v4/platform/notifications) |
| **Admin ⇄** | Переключатель поверхности. Виден только ролям с Admin-доступом. Состояние (где был) запоминается |
| **Профиль 👤** | Личные настройки, настройки уведомлений, тема, выход |

### Сайдбар Workspace — только проект

Сайдбар относится к текущему проекту и существует только в нём ([D-011](/v4/decisions)). **Home всегда без сайдбара** — даже при переходе из проекта: Home глобален, экран занимает всю ширину; очередь дел по всем моим проектам + карточки моих проектов. Клик по проекту, элементу очереди или свитчер → проектный режим с сайдбаром. Вернуться в Home всегда можно кнопкой 🏠 в хедере.

**Проектный режим** (внутри проекта):

| Раздел | Бейдж |
|---|---|
| [Dashboard](/v4/workspace/dashboard) | — |
| [Client](/v4/workspace/client) | требует внимания: конфликты материалов, документы на аппруве |
| [Strategies](/v4/workspace/strategies) | гейты на аппруве |
| [Leads](/v4/workspace/leads) | — |
| [Campaigns](/v4/workspace/campaigns) | предупреждения (ящики у лимита) |
| [Inbox](/v4/workspace/inbox) | непрочитанные / Reply Queue |
| [Meetings](/v4/workspace/meetings) | — |
| [Tasks](/v4/workspace/tasks) | overdue + today |
| ⚙️ [Settings](/v4/workspace/settings) — внизу, отдельно | алерты каналов |

**Рельса при внутренней панели ([D-010](/v4/decisions)).** Модули со своей панелью (Inbox — треды и очереди, Settings — разделы, Leads — аудитории) автоматически сжимают глобальный сайдбар в рельсу иконок 54px: бейджи сохраняются, узнаваемость обеспечивает кит ([D-009](/v4/ui-kit)) — у каждого модуля своя иконка. Раскладка: рельса → панель модуля → контент → опционально Details. На «плоских» экранах (Home, Dashboard, Client) — полный сайдбар; вручную свернуть можно всегда, состояние запоминается на пользователя.

### Элементы поверх любого экрана

| Элемент | Поведение |
|---|---|
| **[Company & Contact Overlay](/v4/workspace/overlay)** | Открывается кликом по компании/контакту откуда угодно, не теряя текущий экран. Закрытие — Esc или клик мимо |
| **Системные баннеры** | Полоса под хедером в контексте проекта: «Проект на паузе», «Инвойс просрочен», «Ящик отключён». Один баннер — самый критичный, остальные счётчиком |
| **Toasts** | Подтверждение действий; для обратимых — с кнопкой **Undo** (10 сек) |
| **Шорткаты** | ⌘K поиск, Esc закрыть оверлей; полный список — по `?` |

## Правила и эдж-кейсы

| Ситуация | Поведение |
|---|---|
| У пользователя один проект | Логин всё равно ведёт в Home (решение 2026-07-15): очередь дел всегда на одном и том же месте. Свитчер показывает один пункт |
| Ask AI предлагает действие за пределами прав | Действие не выполняется; ассистент говорит, у кого есть право, и может отправить запрос этому человеку |
| У пользователя ноль проектов | Home пуст — спецблоков для менеджмента нет. Delivery-роли: «Попросите добавить вас в проект»; AS/AD — CTA создания; Ops/C-level — подсказка «Проекты — в Admin → Projects» |
| Админ не в команде проекта хочет его посмотреть | Только через [Admin → Projects](/v4/admin/projects) → клик по проекту. Проект открывается **read-only** с баннером «Вы не в команде проекта». В свитчере и Home чужие проекты не появляются |
| Убрали из проекта, пока он в нём работал | Немедленный редирект в Home с сообщением; несохранённый черновик (письмо, заметка) сохраняется в драфты |
| Deep link в проект без доступа | Экран 403 с кнопкой «Запросить доступ» (уведомление AS проекта) |
| Уведомление о сущности из недоступного проекта | Клик → то же 403; уведомление не раскрывает содержимое |
| Draft-проект в свитчере | Показывается с бейджем Draft только создателю; клик продолжает визард |
| Archived-проект | В свитчере отсутствует; доступен только из Admin → Projects, открывается в read-only с баннером |
| Поиск ⌘K | Ищет только в доступных пользователю проектах; результаты из архива помечены |
| Потеря соединения | Глобальный оффлайн-баннер, действия блокируются с сохранением введённого |

## Принятые решения (2026-07-15)

| Решение | Детали |
|---|---|
| Логин всегда ведёт в Home | Даже при одном проекте — очередь дел не должна зависеть от числа проектов |
| Ask AI в хедере — да | Постоянная кнопка, панель с контекстом экрана; детали агента — в [AI & Automation](/v4/platform/automation) |
| Мобильной версии нет | Продукт — desktop web. Минимальная ширина ~1280px; уже — предупреждение «откройте на десктопе». Срочные события доходят через Slack-уведомления ([карта](/v4/platform/notifications)) |
