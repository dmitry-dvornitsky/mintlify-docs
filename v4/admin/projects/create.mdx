Создание проекта — линейный визард (наследник Setup из V3): один экран — одна задача — одна кнопка. Обязательны только шаг 1 и шаг 2; остальные можно пропустить и закрыть позже — [Home](/v4/workspace/home) будет напоминать чеклистом.

## Кто использует

AS и AD (и выше). SDR не создаёт проекты — попадает в готовый по приглашению на шаге 3.

## Как работает

| Шаг | Что происходит | Можно пропустить |
|---|---|---|
| 1. Проект | Название + сайт клиента. AI сразу уходит изучать сайт в фоне | Нет |
| 2. Подписка | Сервис из [каталога](/v4/admin/finance), тип оплаты, инвойс с preview | Нет — без подписки проект не активируется |
| 3. KPI и команда | Число встреч/мес + приглашения по email | Да |
| 4. Каналы | Ящики и LinkedIn; система считает, сколько нужно под KPI; warmup стартует сам | Да |
| 5. Материалы | Drag-and-drop файлов и ссылок клиента | Да |
| 6. Документы | ✦ Generate client docs → 4 документа → Approve ([D-038](/v4/decisions): без принудительного открытия) | Да, но без Approve не запустить кампейны |

## Правила и эдж-кейсы

| Ситуация | Поведение |
|---|---|
| Вышел посреди визарда | Проект сохранён как **Draft**, прогресс не теряется, продолжение с того же шага |
| Домен уже есть в другом проекте | Предупреждение с ссылкой на существующий проект; создать всё равно можно (renewal, второй продукт) |
| Сайт клиента недоступен / не парсится | Шаг 1 проходит; AI помечает «мало контекста», просит больше материалов на шаге 5 |
| Инвойс отправлен, но клиент не оплатил | Проект активен, работа идёт; просрочка — бейдж и алерт AD ([Finance](/v4/admin/finance)) |
| Приглашённый не принял инвайт | Виден в команде как Pending, можно перепослать или отозвать |
| Удаление Draft | Единственный случай настоящего удаления в продукте наряду с GDPR ([D-007](/v4/decisions)) — черновик без данных, каскада нет |
