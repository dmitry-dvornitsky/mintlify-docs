Полный жизненный цикл: `Draft → Onboarding → Active → (Paused) → Closed → Archived`. Каждый переход отвечает на вопрос «что происходит со связанными данными».

<Warning>Статусная модель этой страницы — спека до прохода «факт vs TODO» ([D-050](/v4/decisions)). Принята новая модель полей состояния ([D-054](/v4/decisions)): Status (Pending / Active / On Hold / Closed + причина) · Phase · Health. Соответствие с Draft/Archived уточним при проходе этого модуля.</Warning>

## Кто использует

AS/AD — пауза, закрытие, переименование, передача. Ops/C-level — архив и разархивирование. Остальные роли статусами не управляют.

## Как работает

| Статус | Что означает | Кто переводит |
|---|---|---|
| Draft | Визард не завершён | Автоматически |
| Onboarding | Визард завершён, первый кампейн не запущен | Автоматически |
| Active | Идёт работа | Автоматически при запуске кампейна |
| Paused | Работа приостановлена | AS/AD вручную |
| Closed | Контракт завершён / churn | AD |
| Archived | Скрыт из рабочих списков, read-only | Ops/C-level |

## Правила и эдж-кейсы

| Ситуация | Поведение |
|---|---|
| **Пауза** | Все кампейны стоп, отправка писем прекращается немедленно. Inbox продолжает принимать входящие (лидам нельзя не ответить). Warmup ящиков продолжается. Подписка/инвойсы — вопрос к AD: система спрашивает при паузе «остановить и биллинг?» |
| **Возобновление** | Кампании НЕ стартуют сами — AS явно выбирает, какие включить (за время паузы лиды могли устареть) |
| **Закрытие (churn)** | Чеклист офбординга: остановить кампейны, финальный инвойс, отключить каналы клиента, закрыть доступ в Portal. Ящики и LinkedIn-аккаунты освобождаются |
| **Архив** | Read-only навсегда: данные и статистика доступны, действия запрещены. Из списков скрыт. Обратимо (unarchive → Closed) |
| **Удаление** | Проект **не удаляется никогда** — hard delete запрещён во всём продукте ([D-007](/v4/decisions)). Archived — конечное состояние; все данные (треды, лиды, встречи, инвойсы) хранятся. Единственные исключения: GDPR-запрос (флоу в [Admin → System](/v4/admin/system)) и Draft без данных |
| **Переименование** | В любой момент; история и ссылки не ломаются (ID неизменен) |
| **Передача AS → AS** | Один экран: новый владелец получает проекты, гейты, задачи; уведомление команде и опционально клиенту |
| **Удалён последний участник** | Запрещено: у проекта всегда есть владелец-AS; сначала передача |
| **Просрочка инвойса 30+ дней** | Авто-пауза кампейнов с алертом AD (порог настраивается в [Finance](/v4/admin/finance)) |
| **GDPR-запрос клиента после churn** | Строгий админ-процесс в [Admin → System](/v4/admin/system): персональные данные стираются, статистика остаётся анонимизированной ([D-007](/v4/decisions)) |

## Чеклист функционала

- [ ] Статусная модель с автоматическими и ручными переходами
- [ ] Пауза: немедленный стоп отправки, входящие живут, диалог про биллинг
- [ ] Возобновление с явным выбором кампейнов
- [ ] Закрытие с чеклистом офбординга (кампейны, инвойс, каналы, Portal)
- [ ] Освобождение ящиков/LinkedIn при закрытии
- [ ] Архив: read-only, скрытие из списков, unarchive
- [ ] Hard delete проекта отсутствует (D-007): Archived — конечное состояние, все данные хранятся
- [ ] Переименование без потери истории
- [ ] Передача проекта: гейты, задачи, треды — одним действием
- [ ] Защита от проекта без владельца
- [ ] Авто-пауза по просрочке инвойса с настраиваемым порогом
- [ ] GDPR-удаление данных по запросу
