Проект — корневая сущность платформы: один проект = один контракт с клиентом. **В навигации Workspace раздела Projects нет** — delivery-команда попадает в свои проекты через [Home](/v4/workspace/home) и project switcher в [хедере](/v4/layout). Здесь, в Admin, живёт управление портфелем: список, создание, статусы, архив.

## Кто использует

| Роль | Что делает |
|---|---|
| **AS / AD** | Создаёт проекты, управляет жизненным циклом |
| **Ops / C-level** | Видят весь портфель; архив и разархивирование — только они |
| **SDR / LR / EDS / CW** | Сюда не заходят: работают в проектах через Home |

Полная матрица — в [Ролях и правах](/v4/platform/roles).
