Home — единственный экран вне проектного контекста (см. [Layout](/v4/layout)). Раздела Projects в Workspace нет: **мои проекты живут здесь**, вход в проект — клик по карточке или по элементу очереди.

## Планируемые подстраницы

- Очередь дел — агрегат по всем моим проектам: AI-черновики на аппрув, гейты, просроченные задачи, неотвеченные ответы; клик открывает элемент в контексте его проекта
- Мои проекты — карточки: stage, KPI-прогресс, алерты; для AS/AD — кнопка «+ New project» (ведёт в [визард](/v4/admin/projects/create))
- KPI-прогресс — бар месяца по каждому проекту, помесячная корректировка
- Онбординг-чеклист — для проектов в статусе Draft/Onboarding (наследие V3 Setup)

## Вопросы к проработке

- Приоритизация очереди: что выше — гейт, старый ответ лида или просроченная задача; настраивается ли
- Пустая очередь — «всё сделано» vs рекомендации следующих действий

<Note>
Решено (2026-07-15):
- Логин **всегда** ведёт в Home, даже при одном проекте — см. [Layout](/v4/layout).
- Home показывает только **свои** проекты и **свою** очередь. Для тех, кто не назначен ни в один проект (C-level, Ops), Home пуст: empty state без спецблоков. Delivery-роли видят «Попросите добавить вас в проект», роли с Admin-доступом — подсказку «Проекты — в [Admin → Projects](/v4/admin/projects)». Просмотр чужого проекта — только через Admin → Projects, в read-only.
</Note>
