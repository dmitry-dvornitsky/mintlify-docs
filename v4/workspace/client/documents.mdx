Четыре коротких документа фиксированной структуры, сгенерированные из [материалов](/v4/workspace/client/materials) и сайта. Из них AI строит стратегии, квалифицирует лидов и пишет письма.

| Документ | Отвечает на вопрос |
|---|---|
| **О клиенте** | Кто это, что за бизнес, чего ждёт от нас |
| **Продукт** | Что продаём, почём, чем лучше конкурентов, возражения |
| **Кому продаём** | Целевые компании и люди (ICP) |
| **Что говорим** | Боль, оффер, тезисы, чего избегать, кейсы |

Секции каждого документа, источники данных и промпты генерации — на отдельной странице: [Схемы документов и промпты](/v4/workspace/client/document-schemas).

## Кто использует

AS генерирует и аппрувит; CW правит тексты; SDR и LR читают. Клиентского аппрува в V4 нет ([D-022](/v4/decisions)) — вернётся отдельной фичей вместе с Portal-функциональностью.

## Как работает

У каждого документа: **Edit** (редактор), **✦ Regenerate** (с полем «что поменять») и статус. Жизненный цикл версии:

```
Generated (v1, Draft) → Approved ──▶ правка/regenerate ──▶ v2 Draft → Approved
                                        │
                          AI продолжает использовать
                          последнюю Approved-версию
```

Ключевое правило: **AI всегда работает только с последней одобренной версией**. Правка создаёт новую Draft-версию — до её аппрува ничего в продукте не меняется.

Documents — **первая таба и дефолт** модуля Client: чтение (CW, SDR, LR — ежедневно) чаще загрузки материалов (эпизодически, AS).

### Документ — отдельная страница, не попап

Просмотр и редактирование документа — **полноценная страница** с URL (`/client/documents/product`): контент длинный, работа с ним требует фокуса, ссылку на документ шарят в Slack и задачах. Три режима одной страницы:

| Режим | Когда | Что на экране |
|---|---|---|
| **Read** (дефолт) | Открытие по клику/ссылке | Контент Approved-версии, сайдбар: версии, зависимости |
| **Edit** | Кнопка Edit (создаёт или открывает общий Draft) | Секционный редактор + сайдбар: структура секций, presence, источники |
| **Diff review** | После Regenerate или у Draft на аппруве | Дифф против Approved + Accept/Reject, версии, зависимости |

### Read — самый частый режим

Ежедневный сценарий SDR/CW: открыть и взять. Поэтому в Read:

- **Source refs `[mN]`** у каждого пункта — наведение показывает источник, клик открывает материал в [оверлее](/v4/workspace/client/materials). `web` — ссылка на скан сайта.
- **Пометки цен public / internal**: SDR цитирует в письмах только public.
- **Копирование секции** одной кнопкой — CW забирает тезисы в письмо без выделения текста.
- **Sources-карточка** предупреждает: «3 новых материала не в этой версии» — прозрачно, насколько документ свеж.

### Редактор документа

Правка — не пустой markdown, а **секционный редактор**:

- Структура фиксирована схемой документа (у «Продукт»: Что продаём · Цены · Отличия от конкурентов · Возражения · Кейсы). Секции нельзя добавлять, удалять или переименовывать — только наполнять. Это контракт с AI: он знает, в какой секции какое знание искать.
- Внутри секции — простой rich text: заголовки, списки, выделение. Без таблиц и вложенных страниц.
- ✦-ассист на секции: улучшить / сократить / переписать из материалов — правит только эту секцию.
- Автосохранение в Draft; **Submit** переводит Draft в статус «на аппруве».
- **Блокировки нет** ([D-021](/v4/decisions)): у документа один общий Draft, редактировать его могут несколько человек одновременно — в редакторе и на карточке виден presence-индикатор «Kate редактирует сейчас».

**Своих документов нет.** Пятый документ создать нельзя — фиксированная четвёрка гарантирует, что AI умеет использовать всё записанное. Нестандартное знание («заметки с QBR», «правила юрлица») загружается в [Materials](/v4/workspace/client/materials): AI прочитает и разложит по четырём документам при следующей генерации.

### Первая генерация

Пока документов нет, таба Documents показывает empty state с одной кнопкой **✦ Generate documents** (активна, когда есть хотя бы один Parsed-материал или спарсенный сайт; обычно запускается ещё из [визарда](/v4/admin/projects/create)).

- Генерация идёт **в фоне**: грид показывает 4 карточки с прогрессом по конвейеру ①→④ ([схемы](/v4/workspace/client/document-schemas)). Ждать на экране не нужно — можно уйти куда угодно.
- По завершении — **уведомление** «Документы готовы — Review 1 of 4» ([карта](/v4/platform/notifications)).
- Падение генерации одного документа не блокирует остальные: карточка с ошибкой + **Retry**, три других генерируются дальше.

### Последовательный ревью — без возвратов в грид

«✦ Update documents», «Regenerate all» и первая генерация заканчиваются одинаково: уведомление открывает **флоу ревью** — `Review 1 of 4`: экран диффа → **Approve & next** → следующий документ. **Skip** откладывает документ (Draft остаётся, вернуться можно из грида). Каждый документ по-прежнему аппрувится отдельно — убраны только четыре возврата в грид между ними.

**Принудительного прочтения нет** ([D-038](/v4/decisions)): Approve доступен сразу, ответственность за прочитанное — на AS. Для последующих версий рабочий режим ревью — дифф: он и есть то, что изменилось.

## Правила и эдж-кейсы

| Ситуация | Поведение |
|---|---|
| Approve без чтения | Не блокируется ([D-038](/v4/decisions)) — принудительного открытия и скролла нет; аппрув — ответственность AS. Ревью последующих версий — диффом |
| Regenerate при существующем ручном Draft | Подтверждение: «Draft v5 с правками Kate будет заменён». После подтверждения Draft заменяется (Draft-исключение D-007), автор получает уведомление. Молчаливой потери правок нет |
| Regenerate, пока кто-то в Edit | Regenerate побеждает: редакторы получают уведомление, Draft заменяется (после того же подтверждения инициатором) |
| Двое правят одну секцию одновременно | Draft общий, автосохранение; последняя запись побеждает. Разные секции сливаются без конфликтов. Presence-индикатор показывает, кто в редакторе |
| Дифф произвольных версий | Недоступен — дифф всегда против **последней Approved**. История версий — для чтения и отката |
| Падение генерации | Карточка документа с ошибкой + Retry; остальные документы конвейера генерируются дальше |
| Правка документа при запущенных кампаниях | Новая версия после Approve влияет только на **новую** генерацию: письма, уже созданные и запланированные, не перегенерируются. Система показывает, какие активные кампании используют документ |
| Существенное изменение (сменился ICP) | После аппрува новой версии AI предлагает список затронутого: стратегии, аудитории, активные кампании — с рекомендацией что пересмотреть. Ничего не меняет сам |
| Regenerate после новых материалов | Дифф против текущей версии: что добавилось/изменилось, AS аппрувит диффом, не перечитывая всё |
| Откат версии | Любую прошлую Approved-версию можно сделать текущей (это создаёт новую версию-копию — история линейна) |
| Draft-версия висит 7+ дней | Напоминание AS: «есть неопубликованная правка» |
| Удаление версий | Нет ([D-007](/v4/decisions)) — вся история версий хранится; Draft-версию можно удалить (исключение для Draft) |
| Клиентский аппрув | В V4 фичи нет ([D-022](/v4/decisions)): аппрувит только AS. Вернётся вместе с Portal-функциональностью |
| «Нужен пятый документ» | Нельзя: четвёрка фиксирована — это контракт с AI. Нестандартное знание → [Materials](/v4/workspace/client/materials), AI разложит его по документам при генерации |
| Секция документа пуста после генерации | Секция остаётся с пометкой «недостаточно материалов» и подсказкой, что загрузить; AI не выдумывает содержимое |

## Чеклист функционала

- [ ] Генерация всех 4 документов одной кнопкой из материалов + сайта
- [ ] Empty state с ✦ Generate documents; генерация в фоне: 4 карточки с прогрессом, уведомление по готовности, Retry у упавшей
- [ ] Последовательный ревью «Review 1 of 4»: Approve & next / Skip, без возвратов в грид
- [ ] Approve без принудительного прочтения (D-038); ревью версий — диффом
- [ ] Read: source refs [mN] с переходом в материал, пометки public/internal у цен, копирование секции, «N новых материалов не в этой версии»
- [ ] Документы в индексе ⌘K; source-чипы AI (Inbox, стратегии) кликабельны → Read
- [ ] Regenerate поверх ручного Draft — только с подтверждением; уведомление автору Draft
- [ ] Фиксированная структура каждого документа (не пустой редактор); кастомные документы и секции запрещены
- [ ] Секционный редактор: rich text внутри секций, ✦-ассист на секцию, автосохранение в Draft, Submit
- [ ] Edit в редакторе, ✦ Regenerate с промптом «что поменять»
- [ ] Версионирование: Draft → Approved, история, кто и когда аппрувил
- [ ] AI использует только последнюю Approved-версию
- [ ] Аппрув диффом после regenerate: Accept all / ✓✕ на отдельном изменении / Reject (закрывает Draft — исключение D-007)
- [ ] Список зависимостей: какие стратегии/кампании используют документ
- [ ] Рекомендации после существенного изменения (без автоправок)
- [ ] Откат на прошлую версию через копию
- [ ] Совместное редактирование одного общего Draft: presence-индикатор, автосохранение, без блокировок (D-021)
- [ ] Напоминание о зависшем Draft
