Таб экрана [Billing](/v4/workspace/settings/billing). Инвойсы по проекту.

## Как работает

Таблица (сорт/фильтр, набор колонок через **View**):

| Колонка | Описание |
|---|---|
| Invoice Title | Сервис, к которому относится инвойс |
| Status | Draft · Created · Paid · Overdue |
| Total | Итоговая сумма |
| Paid | Оплачено (— если нет) |
| Issue Date | Дата выставления |
| Due | Issue + Customer NET |
| Source | **Subscription** (создан платформой) / **Synced** (из Wave) |

**Отправка:** Recurring генерит инвойсы автоматически; **отправка клиенту — вручную** — выделить инвойсы чекбоксом → **Send to Wave** в тулбаре; после отправки инвойс синхронизируется с Wave.

**Статус оплаты — автоматически из Wave.** Оплату ловим **вебхуком** Wave: инвойс сам переходит в Paid (и подтягивается Paid-сумма), без ручной сверки.

## Правила

- Due Date = Issue Date + Customer NET (из [Billing Information](/v4/workspace/settings/billing/information)).
- Инвойс, созданный из Recurring-подписки, имеет Source = Subscription; пришедший из Wave — Synced.
- Paid проставляется по вебхуку Wave, а не вручную.
