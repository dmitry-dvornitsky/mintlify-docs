Persona — именованный набор job titles: определяется один раз, переиспользуется при поиске контактов вместо ручного ввода должностей. Раньше называлось Audiences ([D-071](/v4/decisions)).

## Кто использует

Создание и правка — уровни **1–3**.

## Как работает

Таблица с кнопкой **Add Persona**. Поля формы:

| Поле | Описание |
|---|---|
| Name * | Название, например «C-Level Executives» |
| Description * | Кто эти люди и чем уникальны — AI использует при квалификации: какие тайтлы подходят под описание |
| Job Titles * | Должности тегами (ввод + Enter) |
| Exclude Job Titles | Исключения; предзаполнено: Assistant, Associate, Junior, Intern, Student, Trainee, Office |

**✦ Improve with AI** на поле Description — улучшает формулировку.

**Использование:** persona выбирается при поиске контактов. **First / Secondary Priority живут на persona — под вопросом** ([D-070](/v4/decisions)): приоритет предлагается проставлять в момент поиска, не хранить на сущности.

Редактировать и архивировать persona можно; правка тайтлов не влияет на уже прошедшие поиски.

## Правила

- Description — не месседжинг: полей need/solution у persona нет и не будет ([D-041](/v4/decisions)) — роль-специфику письма AI выводит из документов при генерации.
- Exclude-список предзаполняется дефолтными исключениями.
- Правка/архив persona не переигрывает прошедшие поиски.
