Кто клиент и что мы ему обещали — чисто контрактная страница. Заполняется визардом при [создании проекта](/v4/admin/projects/create), здесь — правится.

Сущности «Продукты» в V4 нет ([D-042](/v4/decisions)): знание о том, что клиент продаёт, живёт в AI-документе [«Продукт»](/v4/workspace/client/documents) (версии, аппрув), конкуренты — ссылками в [Materials](/v4/workspace/client/materials). **Радикально разные продукты одного клиента — это два проекта**: свои документы, стратегии, ящики и KPI.

## Кто использует

AS и AD — полный доступ. Остальная команда — чтение (данные видны также в Project info на [дашборде](/v4/workspace/dashboard)).

## Как работает

**General** — компания: Name, Size, Industry, Country, **Timezone** (определяет окна отправки кампаний), Website, Contact Person. Контракт: Customer Since/To, Pilot (мес.), Recurring (мес.), Contract Specifics. Стадия жизненного цикла меняется автоматически по событиям, с ручным переопределением — [статусная модель](/v4/admin/projects/lifecycle).

**KPI** — простая помесячная таблица ([D-044](/v4/decisions)): **Month + Goal, вносится руками**, месяцы можно создавать вперёд на любой горизонт. Клик по строке месяца — **встречи, засчитанные в KPI этого месяца** ([D-046](/v4/decisions)). Единственная метрика — встречи; засчитываются **Held & confirmed**: проведённые и подтверждённые клиентом (подтверждение проставляет AS по фидбеку — клиентского аппрува в продукте нет, [D-022](/v4/decisions); отметка в audit log).

- **Goal** — руками, **только уровень Admin** ([D-046](/v4/decisions) уточняет D-044): план фиксирует менеджмент, AS/AD видят. Никаких типов периодов, разбивок и carry-over: контракт «30 за 3 месяца» вносится как 6-12-12 (AI в визарде подсказывает разбивку с ramp-up — это просто подсказка, не механика).
- **Result** — автоматически из [Meetings](/v4/workspace/meetings) по definition; встреча считается в месяц проведения.
- Pace, прогноз и обратная воронка — не здесь, а в [Dashboard](/v4/workspace/dashboard): Settings отвечает «что обещали», Dashboard — «успеваем ли».
- К [Billing](/v4/workspace/settings/billing) **не привязан**: подписки и деньги — отдельно, связь только аналитическая (cost per meeting в [Admin · Finance](/v4/admin/finance)).

## Правила и эдж-кейсы

| Ситуация | Поведение |
|---|---|
| Смена Timezone при активных кампаниях | Окна отправки пересчитываются со следующего дня; уже запланированные на сегодня письма уходят по старым окнам. Предупреждение при сохранении |
| Конец пилотного периода | Открытый вопрос — что происходит автоматически. Минимум: алерт AD за 7 дней до конца |
| Правка KPI задним числом | Разрешена (уровень Admin — [D-046](/v4/decisions)), история хранит обе версии; отчёты клиенту, уже отправленные, не пересчитываются |
| Удаление KPI-записи | Только **будущие** месяцы ([D-046](/v4/decisions)); текущий и прошлые не удаляются никогда — история не стирается |
| Пауза проекта | AD правит Goal месяцев паузы руками (обычно 0); автоматики нет |
| Встреча проведена в декабре, подтверждена в январе | Считается в месяц проведения — иначе темп подтверждений управлял бы отчётностью |
| На текущий месяц Goal не внесён | Алерт AD + строка в [Command Center](/v4/admin/command-center); Dashboard показывает «плана нет». Работа не блокируется |
| Смена Website | Системная строка сайта в Materials пересканируется ([D-019](/v4/decisions)), AI предлагает регенерировать документы — сайт лежит в основе всего |
| У клиента появился второй, радикально другой продукт | Не расширяем проект — AD создаёт второй проект клиента ([D-042](/v4/decisions)): свои документы, стратегии, ящики, KPI |

## Чеклист функционала

- [ ] General: правка всех полей, валидация, Timezone с предупреждением
- [ ] Стадия: автопереходы + ручное переопределение с записью в историю
- [ ] KPI: помесячная таблица Month + Goal, ручной ввод, создание месяцев вперёд (D-044); добавление и правка — уровень Admin (D-046)
- [ ] Definition: Held & confirmed; подтверждение — AS, с audit log
- [ ] Result из Meetings автоматически, в месяц проведения
- [ ] Алерт «Goal на текущий месяц не внесён»; алерт AD за 7 дней до конца пилота
- [ ] История правок Goal (кто и когда); удаление — только будущих месяцев (D-046)
- [ ] Drill-down из строки месяца: встречи, засчитанные в KPI (D-046)
- [ ] AI-подсказка разбивки с ramp-up в визарде проекта (только подсказка)
