Библиотека переиспользуемых объектов таргетинга. Всё создаётся **inline из мест использования** (форма стратегии, поиск лидов, оверлей) — сюда приходят управлять библиотекой, а не начинать работу.

## Кто использует

AS и LR — создание и правка. SDR — чтение (и inline-создание списков). DNC пополняют все, удаление из DNC — AS.

## Как работает

**Audiences** — именованные наборы должностей: Name, Description (AI использует для нестандартных тайтлов), Job Titles, Exclude Titles (предзаполнено: Assistant, Junior, Intern…). Используются в поиске контактов.

**Qualification Criteria** — критерии AI-анализа компаний: до 3 Should + до 3 Should not свободным текстом, **✦ Suggest criteria** из ICP-документа, **Accuracy feedback** — доля результатов, подтверждённых людьми: видно, работает ли набор.

**Custom Columns** — кастомные поля компаний/контактов с AI-обогащением: Name, Type, инструкция обогащения. Заполняются пайплайном, bulk-действием или через MCP.

**Do Not Contact** — исключения: Companies (домены) / Contacts (email). Добавление вручную, CSV-импортом и **автоматически** из ответов типа DNC/Unsubscribe. Колонка Source: Manual / Import / Auto.

**Lists** — именованные списки компаний или контактов (не смешиваются). Создаются здесь, из Overlay и bulk-действий; используются как фильтр в Leads и при наборе кампании.

## Правила и эдж-кейсы

| Ситуация | Поведение |
|---|---|
| Удаление аудитории/критериев, используемых в активной стратегии | Предупреждение со списком зависимостей; объект архивируется — стратегия продолжает работать со снапшотом |
| Правка критериев после прошедших анализов | Результаты не пересчитываются автоматически; предлагается перезапуск анализа на затронутых компаниях |
| Добавление в DNC контакта из активной кампании | Немедленное исключение из всех кампаний и остановка запланированных писем; тред в Inbox остаётся (читать можно, писать нельзя) |
| Добавление домена в DNC | Исключаются все контакты домена — текущие и будущие |
| Удаление из DNC | Только AS, с указанием причины (записывается в audit log); Auto-записи (unsubscribe) удалить нельзя |
| Архивирование кастомной колонки | Hard delete запрещён ([D-007](/v4/decisions)): колонка скрывается из таблиц и форм, данные сохраняются; разархивирование возвращает всё |
| Правка инструкции обогащения | Уже заполненные значения не трогаются; новое обогащение — по новой инструкции |
| Архивирование списка | Скрывается только список, не его участники; кампании, набранные из списка, не затрагиваются |
| Дубликат имени объекта | Запрещён в рамках проекта — имена уникальны |

## Чеклист функционала

- [ ] Audiences: CRUD, теги должностей, exclude-предзаполнение, ✦ Improve with AI
- [ ] Qualification Criteria: CRUD, лимит 3+3, ✦ Suggest из ICP, Accuracy feedback
- [ ] Custom Columns: CRUD, 7 типов, AI-инструкция, табы Contacts/Companies
- [ ] DNC: ручное добавление, CSV-импорт, автопополнение из ответов, Source, защита Auto-записей
- [ ] Lists: CRUD, тип (companies/contacts), inline-создание из Overlay и bulk
- [ ] Inline-создание всех объектов из мест использования
- [ ] Только архивирование, hard delete нет (D-007); снапшот для активных стратегий; разархивирование
- [ ] Использование объекта видно из него («где используется» — список стратегий/кампаний)
