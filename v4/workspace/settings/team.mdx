Кто имеет доступ к проекту. Три секции на одной странице: сотрудники Belkins, AI-агенты и клиентский доступ — их нельзя смешивать визуально и по правам.

## Кто использует

AS и AD управляют составом. Увольнение из компании (не из проекта) — только через [Admin → Team](/v4/admin/team).

## Как работает

**Belkins staff** — таблица: Name, Role (роль на этом проекте), Email, Status (Active / Pending + Resend), Workload (на скольких проектах человек, ⚠ при перегрузе — данные из [Admin → Team](/v4/admin/team)). Действия: **+ Add member** (существующий пользователь + роль) и **Invite** (email новому человеку).

**AI-агенты проекта** — какие агенты активны и режим каждого: **ждёт аппрува / автопилот** ([D-039](/v4/decisions); реестр агентов — [AI & Automation](/v4/platform/automation)).

**Client access** — отдельная секция: Email, Role (Client Admin / Client Viewer), Status. **+ Invite client** — приглашение в [Client Portal](/v4/portal).

## Правила и эдж-кейсы

| Ситуация | Поведение |
|---|---|
| Удаление участника из проекта | Диалог передачи: его лиды, треды, задачи и гейты в этом проекте — выбранному преемнику. Без преемника удалить нельзя |
| Удаление последнего AS | Запрещено — у проекта всегда есть ответственный AS; сначала назначить нового |
| Смена роли участника | Применяется мгновенно; текущие назначения (треды, задачи) остаются, но недоступные новой роли действия пропадают |
| Инвайт на email существующего пользователя | Система предлагает Add member вместо дубля |
| Pending-инвайт 7+ дней | Напоминание пригласившему; Resend и Revoke доступны всегда |
| Перевод агента из автопилота в «ждёт аппрува» | Действует немедленно: очередь агента уходит на аппрув; уже выполненные действия не откатываются |
| Отключение агента | Его незавершённые задачи попадают в очередь AS |
| Клиентский доступ при паузе проекта | Сохраняется (клиент видит дашборд с баннером паузы); при закрытии — отключается по чеклисту офбординга ([lifecycle](/v4/admin/projects/lifecycle)) |
| Приглашение клиента с доменом, не совпадающим с доменом клиента | Предупреждение (защита от опечатки и утечки данных чужому) |

## Чеклист функционала

- [ ] Таблица staff: роль на проекте, статус, workload с предупреждением о перегрузе
- [ ] Add member (существующий) и Invite (новый) с выбором роли
- [ ] Resend / Revoke инвайта, напоминание о зависших
- [ ] Удаление участника только с передачей работы преемнику
- [ ] Защита последнего AS
- [ ] Смена роли с мгновенным пересчётом прав
- [ ] Секция агентов: список, режим аппрув/автопилот (D-039), отключение с передачей очереди
- [ ] Client access: invite, роли Admin/Viewer, проверка домена, revoke
