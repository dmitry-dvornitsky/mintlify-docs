Каналы, через которые проект общается наружу. Здоровье каналов платформа мониторит сама; зона ответственности — EDS. Алерты каналов поднимаются бейджем на пункт Settings в [сайдбаре](/v4/layout).

## Кто использует

AS подключает при setup'е; EDS следит за здоровьем; SDR каналы не настраивает.

## Как работает

**Mailboxes** — Email, Provider (Google OAuth / Microsoft OAuth / SMTP), **Health 0–100** (bounce rate, spam-репорты, blacklists, возраст домена), Usage (отправлено сегодня / лимит), Warmup-статус. Новый ящик автоматически прогревается — лимит растёт по графику; кампейны не шлют с неготовых ящиков. Дневной лимит можно **править вручную** — это потолок, прогрев растит лимит до него ([D-047](/v4/decisions)). У ящика видно **в каких кампейнах используется** (с переходом) и **историю отправок** по дням ([D-047](/v4/decisions)).

**LinkedIn** — аккаунты через Unipile: Owner, Status (Active / Reconnect needed / Restricted), дневные лимиты инвайтов и сообщений. Лимиты защищают аккаунты от блокировок.

**Calendars** — Google / Microsoft (OAuth), Calendly (API key). Минимум один нужен для [Meetings](/v4/workspace/meetings); питают Propose slots в Inbox.

**Slack** — канал проекта + матрица «событие → включено» per-project (встречи, гейты, паузы кампейнов, сообщения клиента).

## Правила и эдж-кейсы

| Ситуация | Поведение |
|---|---|
| Health ящика ниже порога | Ящик автоматически ставится на паузу во всех кампейнах, объём перераспределяется на здоровые ящики, алерт EDS |
| Ящик отвалился (пароль, отзыв OAuth) | Статус Reconnect needed, отправка стоп, алерт EDS + AS; входящие с момента реконнекта дотягиваются |
| Архивирование ящика с активными кампейнами | Подтверждение-согласие «опасное действие» ([D-047](/v4/decisions)): ответы в треды этого ящика перестанут приходить; контакты, чей сиквенс шёл с него, встают на паузу (стики — D-031); новые первые касания уходят с других ящиков, при их отсутствии — кампейн на паузу |
| Архивирование ящика | Hard delete запрещён ([D-007](/v4/decisions)): ящик архивируется, история писем и треды остаются навсегда; подключение (OAuth-токен) отзывается |
| Ящик и кампейны | Один ящик обслуживает **несколько кампейнов** своего проекта ([D-008](/v4/decisions)): дневной лимит общий по ящику, кампейны делят его между собой |
| LinkedIn Restricted | Аккаунт исключается из кампейнов автоматически, алерт; возврат — вручную после проверки EDS |
| Отзыв календаря | Propose slots и букинг ломаются → немедленный алерт AS/SDR, забуканные встречи не пропадают |
| Один ящик в двух проектах | Запрещено: ящик принадлежит одному проекту (лимиты и репутация не делятся) — [D-008](/v4/decisions) |
| Смена Sender name | Применяется к новым письмам; в активных тредах имя не меняется |
| Все ящики проекта нездоровы | Кампании на паузе, баннер в проекте, алерт AS + EDS + Command Center |
