Настройки проекта — нижний пункт проектного [сайдбара](/v4/layout). Принцип V4: **частое настраивается по месту** (inline из форм стратегий, кампейнов, оверлея), Settings — библиотека переиспользуемых объектов и редкие настройки.

Layout — [паттерн рельсы D-010](/v4/decisions): глобальный сайдбар сжат в иконки, панель модуля — навигация по разделам (видимость по ролям) с поиском «Find a setting…» и алерт-бейджами; Details-панели нет — контент занимает всю ширину.

## Кто использует

| Раздел | Кто работает |
|---|---|
| [General](/v4/workspace/settings/general) | Уровни 1–2 |
| [KPI](/v4/workspace/settings/kpi) | Правка 1–2, смотрят все |
| [Команда](/v4/workspace/settings/team) | AS (уровни 1–2) |
| [Mailboxes](/v4/workspace/settings/mailboxes) · [LinkedIn](/v4/workspace/settings/linkedin) · [Calendars](/v4/workspace/settings/calendars) | Подключение и настройка — 1–3 |
| [Notifications](/v4/workspace/settings/notifications) | Slack-канал и алерты проекта |
| [Personas](/v4/workspace/settings/personas) · [Columns](/v4/workspace/settings/columns) · [DNC](/v4/workspace/settings/dnc) · [Lists](/v4/workspace/settings/lists) | AS, LR; SDR — чтение |
| [Billing](/v4/workspace/settings/billing) · [Usage](/v4/workspace/settings/usage) | Admin (1) + AS (2) — остальные не видят |
| [Reply Templates](/v4/workspace/settings/reply-templates) · [Labels](/v4/workspace/settings/labels) | Inbox: назначают SDR+, создают 1–3 |

Недоступные роли разделы не видят вовсе (не disabled). Полная матрица — [Роли и права](/v4/platform/roles).
