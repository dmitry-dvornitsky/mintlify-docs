Поиск лидов — **одна кнопка на стратегии**. Двухфазность «компании → контакты» сохранена в механике (отсев плохих компаний до траты кредитов на контакты), но убрана из UX: юзер не настраивает шаги и не проходит паузы — он запускает конвейер и ревьюит результат ([D-025](/v4/decisions), [D-097](/v4/decisions)).

## Конвейер

```
[✦ Find leads]
      │           всё в фоне (паттерн D-018) — можно уходить
      ▼
① Estimate (Apollo)   «под критерии существует ~2 400 компаний»
② База проекта        340 подходящих уже есть: 120 New · 80 Contacted · 80 Active · 60 DNC
③ Apollo              компании → тихая авто-квалификация → контакты только в прошедших
④ Generect (LinkedIn) доиск контактов там, где Apollo не покрыл
      │
      ▼
Уведомление: «Найдено 214 контактов в 180 компаниях — Review»
```

Waterfall по цене ([D-025](/v4/decisions)): каждый следующий источник дороже; вниз спускаемся только за недостающим. База проекта — бесплатна и мгновенна, поэтому переиспользование лидов между стратегиями происходит само. Кто из подходящих в базе уже отрабатывался и чем закончилось — видно по статусу **Usage** ([D-102](/v4/decisions)): шаг ② раскрывает разбивку New / Contacted (с исходами) / Active / DNC ещё до траты кредитов.

**Estimate виден до запуска** — на карточке стратегии, вместе с «в базе уже N». Прогноз стоимости — там же: «~900 новых ≈ N кредитов».

Этапы запуска юзер видит как **таймлайн прогресса** (как трекинг посылки), не как формы. Настройки конвейера (target count, **колонки квалификации**, max contacts per company, источники) — в Show details стратегии; правит AS, LR может **переопределить на конкретный запуск** в рамках заданных AS границ ([D-097](/v4/decisions)) — не трогая сохранённую стратегию.

**Колонки квалификации** ([D-097](/v4/decisions), заменяет опциональный Gate): у стратегии — опциональный список Custom Columns + значения, означающие «подходит» (например, Tech Stack содержит Salesforce). Если задано — конвейер сам прогоняет Enrich по этим колонкам на найденных компаниях и **автоматически** продолжает поиск контактов только по прошедшим — без паузы, без участия человека. Не задано — конвейер идёт напрямую, как в простом сценарии. Разница между «глубоким» и «быстрым» поиском — не ручной шаг, а то, настроена ли квалификация у этой стратегии.

**Find more** — та же кнопка после первого запуска: «Find 200 more», skip existing автоматически, никаких форм.

Правила набора:

| Правило | Поведение |
|---|---|
| **Usage — статус использования** ([D-102](/v4/decisions)) | На каждом контакте в ревью: **New** (не был в кампейнах) / **Active** (занят сейчас) / **Contacted** с исходом последнего кампейна (No reply · Replied · Positive · Meeting · Bounced). Вычисляется из истории кампейнов — переиспользование видно сразу, без археологии |
| **Эксклюзивность контакта** ([D-026](/v4/decisions)) | Контакт занят максимум одним активным кампейном. Active видны серыми: «занят Q3 Outbound до 12.08» — добавить нельзя. Кулдауна между кампейнами нет |
| **Свежесть — 90 дней** ([D-027](/v4/decisions)) | Контакт устаревает через 90 дней: при наборе кампейна — автоматическая ревалидация (email verify, LinkedIn score); бейдж «stale» в ревью |
| **DNC** | Применяется всегда, до всех источников ([Do Not Contact](/v4/workspace/settings/dnc)) |
| **Квалификация** | Не отдельный вердикт — значения [Custom Columns](/v4/workspace/settings/columns) на компании/контакте ([D-074](/v4/decisions)): факт общий, не зависит от стратегии; каждая стратегия применяет свой фильтр по колонкам |
| **Атрибуция** | «Источник лида» (какая стратегия/запуск нашла) — отдельное поле; метрики считаются по кампейну, который вёл коммуникацию |
| **Контакт → аудитория** | Контакт хранит аудиторию, по которой найден ([D-041](/v4/decisions)): при генерации письма AI берёт роль-специфику из документов именно для неё — без гадания по тайтлу |

## История запусков

Внутри стратегии — плоский список запусков (замена Requests из V1): дата, кто запустил, estimate на момент запуска, найдено/квалифицировано/добавлено, стоимость, статус. Клик — ревью этого запуска (уже обработанные строки — read-only).

**Batch-группировка** ([D-096](/v4/decisions)): запуски с одинаковыми критериями схлопываются в один **batch** — «Find more» с теми же критериями пополняет текущий batch, а не создаёт новую точку. Новый batch появляется только когда критерии стратегии менялись между запусками. Это нужно, чтобы на [шаге 2 создания кампейна](/v4/workspace/campaigns/create) можно было выбрать «контакты именно из этой партии» — и понять разницу между партиями по тому, что в критериях изменилось, а не по голой дате.

Оценка рынка на каждом запуске/batch — **приблизительный контекст на момент этого запуска**, не сквозная цифра, обязанная сходиться между всеми batch стратегии ([D-099](/v4/decisions)). Если Batch 2 расширяет критерии Batch 1 (например, шире company size), их оценки не примиряются математически — batch решает свою настоящую задачу (группировка + доливка + выбор группы для кампейна), а не бухгалтерию по рынку.

**Детальная batch** ([D-100](/v4/decisions), раздел **Batches** на детальной стратегии — на одном уровне с Campaigns, не тихая ссылка): клик по batch открывает воронку конвейера — Estimate → уже в базе (skip) → отправлено в Apollo → прошли колонки квалификации → найдено контактов (Apollo + Generect) → прошли DNC → свободны (эксклюзивность) → добавлено в базу. На каждой стадии виден и отвал, и причина (квалификация/DNC — реальный отсев; эксклюзивность — не отсев, а занятость другим кампейном). Кнопка «Открыть в Leads →» ведёт в [Leads](/v4/workspace/leads) с фильтром по этому batch — отдельной таблицы компаний/контактов здесь нет, тот же принцип, что для [Lists](/v4/workspace/settings/lists) ([D-082](/v4/decisions)).

## Правила и эдж-кейсы

| Ситуация | Поведение |
|---|---|
| Estimate сильно меньше цели (нужно 200, существует 80) | Предупреждение до запуска + AI предлагает ослабить критерии (через Change с диффом) |
| Найдено 0 новых | Итог с причинами: «всё уже в базе / отсеяно квалификацией / DNC» + рекомендации |
| Квалификация отсеяла почти всё | Алерт с примерами отклонённых и причинами; предложение поправить критерии |
| Все подходящие в базе заняты кампейнами | Видно в ревью: «120 подходящих Active» — честная картина ёмкости сегмента |
| Сегмент выработан (почти все Contacted) | Итог показывает разбивку исходов: «160 из 200 уже контактировали: 120 no reply · 40 replied» — решение о повторном заходе или смене сегмента за человеком |
| Запуск упал (провайдер недоступен) | Этап помечен ошибкой, найденное до падения — в ревью, Retry продолжает с места падения |
| Два запуска одновременно (одна стратегия) | Запрещено: кнопка становится «Running…»; разные стратегии — параллельно можно |
| Две стратегии нашли одну компанию | Компания одна в базе, членство — в обеих; контакты делятся по правилу эксклюзивности |
| Лимиты провайдеров исчерпаны | Этап ждёт с обратным отсчётом, уведомление EDS; запуск не падает |

## Чеклист функционала

- [ ] Конвейер: estimate → база → Apollo (компании → колонки квалификации → контакты) → Generect (D-025)
- [ ] Запуск в фоне, таймлайн прогресса, уведомление по готовности (D-018)
- [ ] Estimate + «в базе уже N» + прогноз кредитов — до запуска
- [ ] Find more со skip existing, без форм
- [ ] **Колонки квалификации на стратегии** — тихая авто-фильтрация компаний перед поиском контактов, без ручной паузы (D-097)
- [ ] **LR override на конкретный запуск** — в рамках границ, заданных AS, без правки сохранённой стратегии (D-097)
- [ ] Ревью: группировка по компаниям, вердикты с причинами, выбор, Add selected
- [ ] **Usage-статус на контактах ревью и в разбивке шага ②** — New / Active / Contacted с исходом (D-102)
- [ ] Эксклюзивность контакта: один активный кампейн, серые занятые (D-026)
- [ ] Ревалидация контактов старше 90 дней при наборе (D-027)
- [ ] Квалификация — значения Custom Columns, общие вне зависимости от стратегии (D-074)
- [ ] Поле «источник лида» отдельно от метрик кампейна (D-027)
- [ ] История запусков со стоимостью и переходом в ревью; batch-группировка по критериям (D-096)
- [ ] **Детальная batch** — воронка по стадиям конвейера с причиной отвала на каждой; переход «Открыть в Leads →» с фильтром по batch, без дубль-таблицы (D-100)
- [ ] Cancel run с сохранением найденного; Retry с места падения
- [ ] Учёт стоимости запусков в [Admin → Finance](/v4/admin/finance)
