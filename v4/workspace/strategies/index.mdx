Стратегия — playbook одного сегмента: **кому** пишем (компании и люди), **что** говорим и **как** ищем лидов. AI собирает стратегии из [документов клиента](/v4/workspace/client/documents), человек выбирает и правит словами. Кампейн без стратегии невозможен.

Ключ модели ([D-024](/v4/decisions)): **лиды принадлежат проекту, а не стратегии**. Стратегия — линза над общей базой: помечает своих лидов членством, компания может состоять в нескольких стратегиях. Переиспользование лидов между стратегиями — не фича, а следствие модели: [поиск](/v4/workspace/strategies/find-leads) всегда начинается с базы проекта.

Наполнение базы — действие стратегии, двумя путями ([D-091](/v4/decisions)): **✦ [Find leads](/v4/workspace/strategies/find-leads)** (конвейер по критериям) и **[Import](/v4/workspace/strategies/import-leads)** (загрузка готового CSV/LinkedIn-списка) — оба живут здесь, не в [Leads](/v4/workspace/leads), который остаётся чистым просмотром базы.

## Кто использует

| Роль | Что делает |
|---|---|
| **AS** | Создаёт, правит, паузит, архивирует |
| **LR** | Запускает поиски лидов, ревьюит находки |
| **SDR, CW** | Читают: сегмент и месседжинг — контекст для писем |

## Как работает

### Список — карточки с живыми метриками

У каждой стратегии: статус, Leads / Кампейны / Reply rate / Meetings, estimate рынка («~2 400 компаний под критерии» — считается через Apollo при создании и по запросу) и главное действие — **✦ Find leads** (после первого запуска — **Find more**).

### Создание: AI предлагает — человек выбирает

Кнопка **✦ Suggest new strategies** генерирует **три карточки** из Approved «Who we sell to» и «What we say» (наследие V3): название сегмента, кому пишем словами, «почему сработает» со ссылкой на материалы, estimate рынка, ⭐ на рекомендованной. С каждым разом AI умнее — учитывает накопленные результаты («похожа на CFO UK — 7 встреч, но сегмент шире»).

**Describe your own** — своя идея словами («клиент просит финтех в Германии»), AI собирает из неё такую же карточку на выбор.

Кнопка **Use strategy** материализует карточку в полную сущность: Goal, Company ICP, Contact ICP (+Persona), Messaging, Personalization hook — всё заполнено AI, всё лежит под **Show details**.

### Детальная: три блока + Show details

Простой слой — три блока с правкой словами (V3 Change): **Каким компаниям** · **Каким людям** · **Что говорим**. Правка → AI показывает дифф «было → стало» → подтверждение. Гибкий слой — **Show details**: полная форма, **колонки квалификации** и настройки конвейера поиска. Один источник данных, два уровня видимости — рассинхрона нет.

**Создаёт и правит стратегию — AS.** LR может **переопределить критерии на конкретный запуск** [Find](/v4/workspace/strategies/find-leads)/[Import](/v4/workspace/strategies/import-leads) в рамках заданных AS границ, не трогая сохранённую стратегию ([D-097](/v4/decisions)) — так критерии, озвученные AS устно, не требуют передачи прав на саму стратегию.

Правила блока «Каким компаниям»:

- **Колонки квалификации** ([D-097](/v4/decisions)) — опциональный список [Custom Columns](/v4/workspace/settings/columns) + значения, означающие «подходит». Если задано — [Find](/v4/workspace/strategies/find-leads) и [Import](/v4/workspace/strategies/import-leads) компаний сами фильтруют перед поиском контактов, без ручной паузы. Пусто — конвейер идёт напрямую.

Правила блока «Каким людям» ([D-041](/v4/decisions)):

- Это **ссылки на персоны** из [библиотеки](/v4/workspace/settings/personas) — персона переиспользуется между стратегиями, тайтлы правятся в одном месте.
- **Need/solution ролей не прописываются**: AI выводит их при генерации писем из документов («Кому продаём» — за что роль отвечает, «Что говорим» — тезисы по ролям) × угол этой стратегии. Контроль — сэмплы перед запуском (D-030).
- **Опциональная заметка по аудитории** — свободный текст-override для исключений: «для CFO здесь не упоминать цену, давить на сроки». Пустая по умолчанию.
- Опциональный **лейбл продукта** — тег для мультипродуктовых клиентов ([D-042](/v4/decisions)): группирует вклад стратегий по продукту в Dashboard и отчётах. Радикально разные продукты — отдельные проекты, не лейблы.

### Batches

Раздел детальной страницы, на одном уровне с Campaigns — список [batch](/v4/workspace/strategies/find-leads) этой стратегии, не тихая ссылка под блоком «Каким компаниям». Клик по строке открывает детальную batch: воронка конвейера (где и почему отвалились компании/контакты — квалификация, DNC, эксклюзивность) + переход «Открыть в Leads →», который ведёт в [Leads](/v4/workspace/leads) с фильтром по этому batch (без отдельной дубль-таблицы компаний/контактов — тот же принцип, что для [Lists](/v4/workspace/settings/lists), [D-082](/v4/decisions)).

| Batch | Diff | Контакты | Персоны | Дата |
|---|---|---|---|---|
| Batch 3 · текущий | + DACH region | 214 | CFO 130 · Finance Director 84 | 20.07 → сейчас |
| Batch 2 | Company size 11–50 → 11–80 | 180 | CFO 95 · Finance Director 85 | 10.07 → 15.07 |
| Batch 1 | исходные критерии | 200 | CFO 112 · Finance Director 68 | 28.06 → 05.07 |

### Campaigns

Раздел детальной страницы — список кампейнов, запущенных на этой стратегии (одна стратегия может обслуживать несколько кампейнов одновременно — разные типы, разные волны или группы). Клик по строке открывает кампейн в [Campaigns](/v4/workspace/campaigns); отдельной детальной здесь нет — только список.

| Name | Type | Progress | Reply / Pos / Mtgs | Status |
|---|---|---|---|---|
| Q3 Outbound | ✦ AI | 612/856 · ~28.07 | 41 / 12 / 5 | Active |
| Precision CFO | Static | 120/400 · ~11.08 | 8 / 2 / 1 | Paused |

**+ New campaign** — тот же визард [создания кампейна](/v4/workspace/campaigns/create), шаг 1 предзаполнен этой стратегией.

### Жизненный цикл

| Действие | Поведение |
|---|---|
| **Pause** | Не предлагается для поисков и новых кампейнов; активные кампейны продолжают идти |
| **Duplicate** | Копия для соседнего сегмента; AI сразу предлагает, что изменить |
| **Archive** | Только без активных кампейнов — иначе блок с CTA «останови или заверши». История лидов и кампейнов сохраняется ([D-007](/v4/decisions)) |
| **Правка** | Не трогает активные кампейны (работают на снапшоте) и найденных лидов — влияет только на будущие поиски и кампейны ([D-028](/v4/decisions)) |

## Правила и эдж-кейсы

| Ситуация | Поведение |
|---|---|
| Больше 5 активных стратегий | Soft-limit: предупреждение «фокус размывается», создать всё равно можно ([D-028](/v4/decisions)) |
| Сменился ICP-документ | Стратегии не меняются сами; алерт «Show affected» из [Documents](/v4/workspace/client/documents) ведёт сюда с рекомендациями |
| Стратегия слабеет (reply rate ниже порога 2 недели) | Карточка в Home: «изменить messaging / предложить новые сегменты» — решение за человеком |
| Duplicate стратегии с лидами | Копируются критерии и месседжинг; лиды и история поисков — нет (лиды и так в базе проекта) |
| Архив стратегии | Членство лидов сохраняется в истории; лиды остаются в базе проекта нетронутыми |
| Правка «Каким компаниям» при найденных лидах | Найденные не трогаются и не помечаются — критерии влияют только на будущие поиски |

- [ ] Список карточек: статус, live-метрики, estimate, Find leads / Find more

- [ ] Describe your own → карточка на общих правах
- [ ] Use strategy → полная сущность (Goal, Company ICP, Contact ICP + Persona, Messaging, Hook)
- [ ] Детальная: 3 блока, правка словами с диффом, Show details с полной формой
- [ ] **Batches** — раздел со списком batch этой стратегии, на уровне Campaigns; клик → воронка отвала + переход в Leads с фильтром ([D-100](/v4/decisions))
- [ ] **Campaigns** — раздел со списком кампейнов этой стратегии + **+ New campaign** ([D-098](/v4/decisions))
- [ ] «Каким людям»: ссылки на аудитории библиотеки, опциональные заметки-override, без прописанных need/solution (D-041)
- [ ] «Каким компаниям»: колонки квалификации — тихая авто-фильтрация в Find/Import, без ручного Gate (D-097)
- [ ] LR override критериев на конкретный запуск в границах, заданных AS, без правки сохранённой стратегии (D-097)

- [ ] Estimate рынка через Apollo: при создании, по клику, дата оценки
- [ ] Членство лидов: компания в N стратегиях, база — проектная (D-024)
- [ ] Pause / Duplicate (с AI-подсказкой) / Archive (блок при активных кампейнах)
- [ ] Правка не трогает кампейны (снапшот) и найденных лидов (D-028)
- [ ] Soft-limit 5 активных, предупреждение
- [ ] Карточка «стратегия слабеет» в Home
- [ ] Чипы версий документов-источников с переходом в Read
