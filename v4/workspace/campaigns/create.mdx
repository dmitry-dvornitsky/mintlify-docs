Создание кампейна — не написание текстов с нуля: AI собирает сиквенс из угла стратегии, человек выбирает лидов и подтверждает. Визард из восьми шагов ([D-048](/v4/decisions)): один экран — одна задача; вход с кнопки «Новый кампейн» на стратегии предзаполняет шаг 1.

## Шаги визарда

| Шаг | Что происходит |
|---|---|
| **1 · Стратегия и тип** | Выбор стратегии + тип: **✦ AI Generated** или **Static** ([D-029](/v4/decisions)). Тип потом не меняется |
| **2 · Контакты** | Набор из лидов стратегии: свободные (эксклюзивность D-026), фильтры, число. Stale-контакты уходят в ревалидацию (D-027) |
| **3 · Сиквенс** | Написание: AI предлагает структуру и содержимое шагов из угла стратегии — правится в редакторе (ниже) |
| **4 · Превью сиквенса** | AI: **сэмплы** на 5–10 реальных лидах (Gate — без просмотра дальше нельзя). Static: рендер на лидах + **отчёт по переменным** |
| **5 · Ящики** | Пул чекбоксами: health, warmup, вклад — механика на [странице Sending](/v4/workspace/campaigns/sending) |
| **6 · LinkedIn** | Аккаунты для LinkedIn-касаний сиквенса: владелец, лимиты, статус. Шаг **пропускается автоматически**, если LinkedIn-касаний в сиквенсе нет |
| **7 · Настройки** | Дни и окна отправки (таймзона лида), приоритет и min/max, регулятор автономии ([D-037](/v4/decisions)), meetings automation |
| **8 · Превью кампейна** | Итоговая сводка: сиквенс, набор, каналы, расписание, прогноз завершения + чеклист готовности → **Launch** (право AS) |

## Редактор сиквенса

Общее для обоих типов: шаги (Email / Follow-up в том же треде / LinkedIn touch), интервалы («wait 3 days»), стоп-условия зашиты (ответ, DNC, встреча — останавливают контакта всегда).

### Тип AI Generated: шаг = промпт

<Frame caption="AI-сиквенс: промпт в каждом шаге, сэмплы на реальных лидах справа, регулятор автономии">
  <img src="/v4/images/campaigns/sequence-ai.png" alt="Редактор AI-сиквенса с промптами и сэмплами" />
</Frame>

- Промпт шага описывает задачу письма: «боль кассового разрыва, кейс BuildCo, CTA 15 минут, 60–80 слов». AI пишет **уникальное письмо каждому лиду**: промпт × угол стратегии × роль-специфика из документов (по аудитории лида, [D-041](/v4/decisions)) × персонализация (hook, LinkedIn, новости).
- Промпты шагов AI генерирует сам из угла — CW правит.
- **Сэмплы**: перед запуском AI пишет письма для 5–10 реальных лидов из набора — панель справа, листаются, обязательны для Gate.
- **Автономия** ([D-037](/v4/decisions)): одобренные сэмплы — достаточный контроль, после запуска **дефолт — автопилот**. Регулятор per кампейн для осторожных клиентов: каждое письмо на ревью / первые 5 каждого шага (очередь в Inbox) / автопилот (дефолт).
- Спинтакс не нужен — каждое письмо и так уникально.

### Тип Static: шаг = темплейт

<Frame caption="Static-сиквенс: темплейт с переменными и спинтаксом, рендер на лиде, отчёт о пропущенных переменных">
  <img src="/v4/images/campaigns/sequence-static.png" alt="Редактор Static-темплейта с переменными и спинтаксом" />
</Frame>

- Subject + body; **переменные** `{{first_name}}`, `{{company}}`, любые [custom columns](/v4/workspace/settings/targeting) — с фоллбеками (`{{first_name|there}}`).
- **Спинтакс** `{Hi|Hello|Hey}` — вариативность при фиксированном месседже (deliverability).
- **Отчёт по переменным** до запуска: «у 12 лидов пусто `{{fleet_size}}`» → фоллбек / исключить / обогатить.
- Рендер-превью на реальных лидах из набора.
- AI помогает и здесь: черновики темплейтов из угла стратегии — CW доводит и фиксирует. После запуска тексты не меняются AI никогда.
- Темплейт один на кампейн: если месседж по ролям сильно разный — это **разные Static-кампейны по аудиториям** (набор лидов фильтруется аудиторией), не варианты внутри одного.

## Правила и эдж-кейсы

| Ситуация | Поведение |
|---|---|
| Запуск без просмотра сэмплов | Нельзя — Gate требует открыть сэмплы (паттерн «первого аппрува» D-018) |
| Персонализации нет (пустой hook, нет LinkedIn) | AI пишет из угла без персонализации — честное письмо, а не выдуманные детали (правило документов: не выдумывать) |
| Ревью первых 5: письмо отклонено | Правка промпта → сэмплы заново; счётчик ревью сбрасывается |
| Static: переменная пуста и фоллбека нет | Лид не получает письмо, попадает в отчёт — молча кривой текст не уходит |
| Черновик визарда | Автосохранение; вернуться можно из списка (Draft) |
| Лиды заняли другой кампейн, пока визард открыт | Превью кампейна (шаг 8) перепроверяет эксклюзивность: «8 лидов заняты — исключены» |
| В сиквенсе нет LinkedIn-касаний | Шаг 6 (LinkedIn) пропускается; добавили касание позже — шаг появляется |
| LinkedIn-касания есть, аккаунты не выбраны | Превью кампейна блокирует Launch с указанием шага |
| Правка сиквенса после превью | Сэмплы/рендер устаревают — шаг 4 требует пересмотра перед Launch |

## Чеклист функционала

- [ ] Визард 8 шагов (D-048): тип → контакты → сиквенс → превью сиквенса → ящики → LinkedIn → настройки → превью кампейна; автосохранение черновика, вход со стратегии
- [ ] Шаг LinkedIn: выбор аккаунтов под LinkedIn-касания, авто-скип без них, блок Launch при касаниях без аккаунтов (D-048)
- [ ] Превью кампейна: сводка, прогноз завершения, чеклист готовности; инвалидация превью сиквенса после правок
- [ ] Строгий тип: AI Generated / Static (D-029)
- [ ] AI: промпты шагов из угла, уникальное письмо на лида, сэмплы 5–10 обязательны
- [ ] Автономия: каждое / первые 5 / автопилот (дефолт) — D-037
- [ ] Static: переменные с фоллбеками, спинтакс, отчёт по переменным, рендер-превью
- [ ] AI-черновики темплейтов для Static, фиксация после запуска
- [ ] Стоп-условия зашиты: ответ, DNC, встреча
- [ ] Gate на запуск: сэмплы/рендер + чеклист (ящики, переменные, эксклюзивность)
