Управление отправкой построено на принципе: **юзер управляет пулом и приоритетами, цифры распределяет система** ([D-031](/v4/decisions)). Ручные квоты «кампейн А получает 20 из 50 ящика X» — микроменеджмент, который разваливается при первой паузе ящика.

## Три уровня управления

| Уровень | Кто пользуется | Что делает |
|---|---|---|
| **1 · Пул** | Все | Кампейн подключает ящики чекбоксами; видно health и warmup. Ёмкость кампейна = сумма долей подключённых ящиков |
| **2 · Авто-распределение** | Система | Лимит каждого ящика ([D-008](/v4/decisions)) делится между активными кампейнами пропорционально их сегодняшней потребности. Ящик встал по health → объём перераспределился, кампейны не останавливаются |
| **3 · Ручки для исключений** | AS, EDS | **Приоритет** (High забирает ёмкость первым — «дожать за неделю») и **min / max**: «гарантированно ≥30/день», «не больше 50/день» |

## Жёсткие механики

| Механика | Правило |
|---|---|
| **Стики контакта к ящику** | Весь сиквенс контакта — с одного ящика: фоллоуапы в том же треде, та же подпись. Назначается при первом касании |
| **Ramp-up** | Новый кампейн стартует с ~20% ёмкости и наращивает по графику — deliverability. Виден на прогнозе |
| **Прогноз** | «214 лидов × 4 касания = 856 писем · с текущим пулом ≈ 9 рабочих дней · закончит ~28.07» — до запуска и на карточке |
| **Расписание** | Окна отправки в таймзоне лида, пн–пт, случайный разброс интервалов |
| **Auto-add warmed** | Опция: ящик дозрел в прогреве → сам входит в пул кампейна |
| **Только ответы** ([D-032](/v4/decisions)) | Open-пиксель не ставим: вредит deliverability и врёт (Apple MP). Метрики: sent, delivered, bounce, reply |

## Правила и эдж-кейсы

| Ситуация | Поведение |
|---|---|
| Сумма min активных кампейнов больше ёмкости пула | Конфликт показывается в момент ввода (не при отправке): «min 120 при ёмкости 90» — сохранить нельзя |
| Два кампейна с High | Делят ёмкость поровну между собой, Normal — по остатку |
| Ящик умер (Reconnect) с живыми тредами | Новые первые касания уходят с других ящиков автоматически; **фоллоуапы существующих тредов замирают** до реконнекта (стики). Долгий даун — AS вручную переназначает: новый тред, история сохранена. Уточняет правило Channels о «переезде писем» |
| Ёмкость упала ниже потребности (пауза ящиков) | Прогноз пересчитан, «закончит ~28.07 → ~11.08» + алерт AS: добавить ящики или снизить объём |
| Все ящики кампейна встали | Авто-пауза кампейна + алерт (правило [Channels](/v4/workspace/settings/channels)) |
| Warmup-ящик в пуле | Шлёт свою прогревную долю; полноценный вклад — после Warmed |

## Чеклист функционала

- [ ] Пул чекбоксами: health, warmup, вклад «сегодня N из лимита»
- [ ] Авто-распределение лимитов пропорционально потребности (D-031, D-008)
- [ ] Приоритет High/Normal; min/max с валидацией конфликтов на вводе
- [ ] Стики контакта к ящику на весь сиквенс; правило смерти ящика (фоллоуапы ждут)
- [ ] Ramp-up график, виден в прогнозе
- [ ] Прогноз завершения: до запуска, на карточке, пересчёт при правках
- [ ] Расписание: окна в таймзоне лида, разброс
- [ ] Auto-add warmed (опция)
- [ ] Без open-трекинга; метрики sent/delivered/bounce/reply (D-032)
