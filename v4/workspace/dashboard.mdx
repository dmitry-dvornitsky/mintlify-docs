Главный экран проектного контекста — первый пункт [сайдбара](/v4/layout), сюда попадаешь при входе в проект из Home или через project switcher. Тот же дашборд в упрощённом виде видит клиент в [Client Portal](/v4/portal) — поэтому каждый блок имеет флаг «виден клиенту / нет».

## Кто использует

Вся команда проекта; Sales — read-only; клиент — урезанную версию (без Needs attention и внутренних блоков).

## Как работает

Табы: **Overview** · **Performance** (Metric × Group by × Split by × Date range) · **Campaigns** (таблица с агрегатами и donut Reply Types) · **Report** (архив еженедельных AI-отчётов клиенту с правкой перед отправкой).

Блоки Overview:

| Блок | Что показывает | Клиент видит |
|---|---|---|
| KPI | Встречи против плана + темп (on track / behind / ahead) | Да |
| Funnel | Contacts → Contacted → Replies → Positive → Meetings; клик по ступени → список лидов | Да |
| Channels | Email и LinkedIn: sent, reply rate, acceptance, встречи | Да |
| Active work | Пайплайны с прогрессом, активные кампейны, ближайшие встречи | Частично |
| Needs attention | Гейты, ящики у лимита, старые ответы, просроченные задачи | Нет |
| AI Weekly Summary | Что изменилось и почему; черновик отчёта клиенту | После правки |
| Project info | Карточка из Settings: команда, timeline, регион, бюджет | Без бюджета |

## Правила и эдж-кейсы

| Ситуация | Поведение |
|---|---|
| Проект только создан, данных нет | Блоки показывают onboarding-состояние: чеклист незакрытых шагов визарда вместо пустых графиков |
| KPI не задан | KPI-блок предлагает задать одним кликом, не показывает 0/0 |
| Смена date range | Все блоки синхронно; дефолт — текущий месяц |
| AI Weekly Summary до конца первой недели | Блок скрыт, не генерирует сводку из шума |
| Отчёт отправлен клиенту, потом данные скорректированы | Отправленный отчёт неизменяем (снапшот); правка — только следующего |
| Paused-проект | Дашборд доступен, баннер «Проект на паузе с даты N» |

## Чеклист функционала

- [ ] Блоки Overview: KPI, Funnel, Channels, Active work, Needs attention, AI Summary, Project info
- [ ] Клик по ступени воронки → отфильтрованный список лидов
- [ ] Date range с синхронизацией всех блоков
- [ ] Таб Performance: конструктор графика, сохранение конфигурации
- [ ] Таб Campaigns: агрегаты, переход в кампейн
- [ ] Таб Report: архив отчётов, правка черновика, отправка, неизменяемость отправленных
- [ ] Export дашборда (PDF/CSV) за период
- [ ] Onboarding-состояние вместо пустых графиков
- [ ] Флаг «виден клиенту» на каждом блоке — консистентность с [Portal](/v4/portal)
