Кит для всех макетов V4. Главный принцип — **интерфейс должен работать без текста**: удалите подписи или переведите их на китайский — назначение каждого элемента всё равно читается. Достигается тройным кодированием: у каждого смысла есть **форма + иконка + цвет**, и ни один смысл не передаётся только цветом или только словом.

Правила кита:

| Правило | Что означает |
|---|---|
| **База — grayscale** | Каркас, текст, границы — только оттенки серого. Цвет появляется, только когда несёт смысл |
| **Один цвет = один смысл** | Синий — действие/выбрано/в работе · зелёный — готово/успех · янтарный — ожидание/внимание · красный — ошибка/блок/необратимое · фиолетовый — AI (всегда с ✦) |
| **Один смысл = одна форма** | Круг — человек и статус · квадрат — объект · pill — атрибут · пунктир — черновик/добавить/пусто · заливка — главное · контур — второстепенное |
| **Иконка обязательна** | Везде, где есть навигация или действие. Текст лишь дублирует иконку |
| **Одна залитая кнопка на экран** | Главное действие всегда одно и всегда синее |

## Иконки

У каждого модуля навигации — своя иконка, одна и та же во всём продукте: в сайдбаре, табах, хлебных крошках, уведомлениях. Удалить — только красная, архив — нейтральный (hard delete запрещён — [D-007](/v4/decisions)).

## Статусы, приоритеты, прогресс

Каждый статус — уникальная пара «форма + цвет», различимая и без цвета (для дальтоников): черновик — пунктирный круг, в работе — недозаполненный круг, ожидание — контурный круг с часами, готово — залитый круг с галкой, блок — восьмиугольник («стоп»), архив — квадрат.

## Проверка китайским

Контрольный тест кита: интерфейс с подписями на незнакомом языке. Если непонятно, что делает элемент, — нарушено тройное кодирование и элемент переделывается.

## Чеклист применения

Для каждого нового экрана V4:

- [ ] Цвет использован только семантически (5 смыслов), каркас — серый
- [ ] Ни один смысл не передан только цветом — есть форма или иконка
- [ ] У каждого действия и пункта навигации есть иконка из набора
- [ ] Одна залитая (синяя) кнопка на экран
- [ ] Опасные действия красные и требуют подтверждения
- [ ] У каждого icon-only контрола есть тултип: название действия (+ шорткат); у текстовых кнопок — только при неочевидных последствиях
- [ ] Права ([D-017](/v4/decisions)): целые разделы и модули без доступа — **скрываются**; отдельные действия — **disabled с тултипом** «Только AS может аппрувить» (дизейбл не ломает лэйаут и учит, у кого какие права)
- [ ] Всё, что сделал AI, помечено ✦ на фиолетовом
- [ ] Экран проходит «проверку китайским»
