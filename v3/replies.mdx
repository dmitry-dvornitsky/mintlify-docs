В V1 Inbox — почтовый клиент: треды, фильтры, метки, шаблоны. В V3 главный режим — **очередь**: система сама подаёт ответы по одному и для каждого уже подготовила решение.

## Reply Queue

Каждый входящий ответ лида AI обрабатывает до того, как покажет человеку:

1. **Классифицирует** — Interested / Not interested / Contact later / Out of office и остальные типы из V1.
2. **Готовит черновик ответа** — с учётом всего треда, документов клиента и Messaging стратегии.
3. **Ставит в очередь** — сортировка по ценности: сначала позитивные.

Человек открывает очередь из Home («4 ответа ждут проверки») и работает карточками:

```
┌──────────────────────────────────────────────────┐
│ Anna Miller · CFO, BuildCo          [Interested] │
│ ── переписка ──────────────────────────────────  │
│ Мы: «…кейс StoneWorks: −30% кассовых разрывов»   │
│ Anna: «Звучит интересно. Пришлите детали, и      │
│ можно созвониться на следующей неделе»           │
│ ── AI-ответ ───────────────────────────────────  │
│ «Anna, отлично! Прикрепляю one-pager. По звонку: │
│ удобно вт 15:00 или чт 11:00 (London)? …»        │
│                                                  │
│   [Send]        [Edit]        [Skip]             │
└──────────────────────────────────────────────────┘
```

| Кнопка | Что делает |
|---|---|
| **Send** | Отправить AI-черновик как есть — следующая карточка |
| **Edit** | Поправить текст и отправить |
| **Skip** | Отложить — карточка вернётся в конец очереди |

Слоты в черновике для букинга — реальные свободные окна из подключённого календаря, AI подставляет их сам.

## Что не доходит до человека

Часть ответов AI обрабатывает полностью сам — в очередь они не попадают:

| Тип ответа | Что делает система |
|---|---|
| Out of office | Сдвигает follow-up на дату возвращения |
| Unsubscribe / DNC | Останавливает касания, вносит в DNC-список |
| Hard bounce | Помечает контакт, исключает из кампаний |
| Forwarded | Создаёт нового лида из пересланного контакта, ставит в очередь с пометкой |

## Правила безопасности

- **AI никогда не отправляет содержательный ответ сам.** Каждое письмо лиду проходит через Send человека. Автоматика — только для служебных типов из таблицы выше.
- **Сложное — с пометкой Needs attention.** Если лид задал вопрос про цену, интеграции или что-то, чего нет в документах клиента, AI черновик всё равно готовит, но помечает карточку и объясняет, в чём не уверен.
- **Каждый Edit — обучающий сигнал.** Правки человека сохраняются как фидбек для следующих черновиков этого проекта.

## Свободный режим — вне очереди

Очередь — рекомендуемый путь, но не единственный:

- **All conversations** — все треды таблицей: поиск, фильтры по типу ответа и каналу, статус Open/Closed. Любой тред можно открыть и ответить в обход очереди.
- **Написать первым** — кнопка Compose из карточки лида: ручное письмо или LinkedIn-сообщение любому лиду вне кампании. AI-редактор помогает, но можно писать целиком самому.
- **Пере-классифицировать** — тип ответа, выставленный AI, меняется вручную прямо на карточке (лид «Not interested» на самом деле «Contact later» — исправил, follow-up запланировался).
- **Snooze** — отложить тред до даты («вернёмся после отпуска лида») — карточка вернётся в очередь в срок.

<Note>Классический вид Inbox (треды таблицей, фильтры, поиск) остаётся по ссылке «All conversations» — очередь его не заменяет, а сидит поверх.</Note>
