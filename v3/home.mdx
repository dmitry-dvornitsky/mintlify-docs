Home — единственный экран, который нужно понимать в V3. Он живёт в двух режимах и сам переключается между ними.

## Режим 1 — Чеклист (до запуска первой кампании)

Пока проект не запущен, Home показывает чеклист setup'а. Каждый пункт — ссылка на соответствующий шаг [визарда](/v3/setup), выполненные отмечаются автоматически.

```
┌─────────────────────────────────────────────┐
│  Acme Corp                     Setup: 4 / 6 │
│  ████████████████░░░░░░░░                   │
│                                             │
│  ✓ Проект создан                            │
│  ✓ Подписка активна, инвойс отправлен       │
│  ✓ KPI: 15 встреч/мес · Команда: 3 чел      │
│  ✓ Подключено: 4 ящика, 2 LinkedIn          │
│  ▶ Загрузить материалы клиента       [Open] │
│  ○ Сгенерировать документы                  │
│                                             │
│  Дальше: стратегия → лиды → кампания        │
└─────────────────────────────────────────────┘
```

Правила режима:

- **Один активный пункт.** Следующий невыполненный шаг подсвечен, у него единственная кнопка на экране.
- **Выйти можно в любой момент** — прогресс сохраняется, Home вернёт на то же место.
- **Блокирующих зависимостей минимум:** нельзя генерить документы без материалов и нельзя запускать кампанию без подключённых ящиков. Всё остальное можно проходить в любом порядке.

## Режим 2 — Today (после запуска)

Когда первая кампания запущена, чеклист сменяется очередью дел. Принцип: **пустая очередь = день сделан**.

```
┌─────────────────────────────────────────────┐
│  Acme Corp          KPI июля: 9 / 15 встреч │
│  ██████████████░░░░░░░░  60% · осталось 12д │
│                                             │
│  Сегодня (7)                                │
│  ● 4 ответа лидов ждут проверки    [Review] │
│  ● 2 лида готовы к букингу встречи  [Book]  │
│  ● 1 ящик отключился — переподключи [Fix]   │
│                                             │
│  Всё идёт само (можно не трогать)           │
│  · Кампания «Extra Leads · CFO UK» — 82     │
│    писем уйдут сегодня автоматически        │
│  · AI ищет 40 новых контактов               │
└─────────────────────────────────────────────┘
```

### Что попадает в очередь

Только то, что требует человеческого решения, отсортировано по важности:

| Приоритет | Тип дела | Действие |
|---|---|---|
| 1 | Ответ лида ждёт проверки | Открывает [очередь ответов](/v3/replies) |
| 2 | Лид согласился на встречу | Открывает [букинг](/v3/meetings) |
| 3 | Follow-up: лид молчит N дней после позитивного ответа | AI подготовил follow-up — Approve |
| 4 | Проблема: ящик отвалился, кампания встала, инвойс просрочен | Кнопка Fix с конкретным действием |
| 5 | Лиды закончились — пора искать новых | Кнопка [Find leads](/v3/leads) |

### Блок «Всё идёт само»

Показывает, что система делает без участия человека: отправку писем, поиск контактов, warmup ящиков. Нужен для доверия — новичок видит, что кампания живёт, и не лезет «помогать».

### KPI-бар

Всегда сверху: план встреч месяца (из [Setup → KPI](/v3/setup#шаг-3--kpi-и-команда)), факт (встречи со статусом Completed), сколько дней осталось. Если темп ниже нужного — Home сам добавляет в очередь подсказку: «При текущем темпе KPI не будет выполнен. Предлагаю: найти ещё 200 лидов / запустить вторую стратегию».

<Note>Это ключевое отличие от V1: там человек сам должен понять, что пора искать лидов или что кампания выдохлась. В V3 продукт говорит это сам.</Note>

## Home — советчик, не диспетчер

В свободном режиме очередь Home ничего не блокирует:

- Любой раздел доступен напрямую в любой момент — искать лидов, править письма кампании или менять стратегию можно без «разрешения» очереди.
- Карточки-подсказки («KPI под угрозой», «кампания слабая») можно **Dismiss** — они советуют, а не заставляют. Отклонённая подсказка не возвращается, пока не изменится ситуация.
- Единственные «жёсткие» карточки — реальные поломки: отвалился ящик, встала кампания, просрочен инвойс. Их скрыть нельзя, пока не починено.

Логика проста: во время онбординга продукт ведёт за руку, потому что человек ещё не знает дороги. После запуска — человек главный, Home лишь следит, чтобы ничего не потерялось.
