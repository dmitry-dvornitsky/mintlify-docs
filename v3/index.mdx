V3 — версия Belkins Home для людей **без опыта в аутриче**. Цель: любой человек заходит в продукт первый раз и доводит клиента до забуканных встреч, ни разу не открыв инструкцию.

Это не отдельный продукт, а упрощённый интерфейс поверх той же платформы и данных, что V1. Вся сложность (ICP, сиквенсы, warmup, deliverability) остаётся под капотом — наружу выведены только решения, которые обязан принять человек.

## Принципы интерфейса

| Принцип | Что это значит на практике |
|---|---|
| **Без жаргона** | Не «Company ICP», а «Каким компаниям пишем». Не «Sequence», а «Письма». Термины аутрича переведены на человеческий язык |
| **Нет лабиринта настроек** | Частое настраивается по месту, прямо в текущем экране. Редкое (подписка, KPI, команда, каналы) — раздел [Project](/v3/project) из четырёх простых карточек |
| **Прогресс всегда виден** | Чеклист setup'а, статус кампании, KPI-бар месяца — человек всегда знает, где он и сколько осталось |
| **Безопасно по умолчанию** | Лимиты отправки, warmup, рабочие часы, DNC — система соблюдает сама. Сломать deliverability из интерфейса невозможно |
| **Advanced спрятан, не удалён** | Под каждым AI-результатом есть «Show details» — полная V1-форма для тех, кто понимает, что делает |

## Весь флоу на одной схеме

```
SETUP (один раз, ~30 минут)                DELIVERY (каждый день, ~30 минут)
┌──────────────────────────────┐           ┌──────────────────────────────┐
│ 1. Создать проект            │           │ 7. Выбрать стратегию от AI   │
│ 2. Подписка → инвойс         │           │ 8. Найти лидов (одна кнопка) │
│ 3. KPI + команда             │    ───▶   │ 9. Запустить кампанию        │
│ 4. Ящики + LinkedIn          │           │ 10. Проверять AI-ответы      │
│ 5. Загрузить материалы       │           │ 11. Букать встречи           │
│ 6. Сгенерить документы       │           │        │                     │
└──────────────────────────────┘           │        ▼ повторять до KPI    │
                                           └──────────────────────────────┘
```

Шаги 1–6 — [Setup-визард](/v3/setup), проходится один раз. Шаги 7–11 — рабочий цикл: [Стратегия](/v3/strategy) → [Лиды](/v3/leads) → [Кампания](/v3/campaign) → [Ответы](/v3/replies) → [Встречи](/v3/meetings).

## Две фазы: рельсы, потом свобода

Guided-флоу — это **онбординг, а не клетка**. Как только первая кампания запущена, рельсы убираются:

| | Фаза 1 — Онбординг | Фаза 2 — Свободный режим |
|---|---|---|
| Кто ведёт | Продукт: один следующий шаг, остальное приглушено | Человек: все разделы полностью доступны в любом порядке |
| Роль Home | Чеклист, ворота к следующему шагу | Советчик: рекомендует, но ничего не блокирует |
| Стратегии | Выбор из 3 карточек AI | Сколько угодно стратегий: новые от AI в любой момент, правка, пауза, дубликаты — [Стратегии](/v3/strategy) |
| Изменения | Идём по шагам | Можно менять всё: документы, письма кампаний, KPI, команду, каналы |

Интерфейс при этом **не меняется** — те же простые экраны, карточки и одна главная кнопка. Свобода достигается не добавлением форм, а тем, что каждый экран доступен напрямую и в любой момент.

## Навигация продукта

Всего 7 разделов — вдвое меньше, чем в V1:

| Раздел | Что внутри | Вместо чего в V1 |
|---|---|---|
| **Home** | Чеклист setup'а, после запуска — очередь дел на сегодня и KPI-прогресс | Dashboard + Tasks |
| **Client** | Материалы клиента и AI-документы — [страница](/v3/client) | Insights + Storage |
| **Strategies** | Все стратегии проекта, новые предложения AI — [страница](/v3/strategy) | Strategies |
| **Leads** | Найденные компании и контакты | Leads (без 20 фильтров наружу) |
| **Campaigns** | Кампании: 4 числа на каждую — Sent, Replies, Positive, Meetings | Campaigns |
| **Inbox** | Ответы лидов + очередь AI-черновиков | Inbox |
| **Meetings** | Забуканные встречи и слоты | Meetings |

Плюс иконка **Project** в углу (не в основной навигации): подписка и инвойсы, KPI, команда, каналы — [страница](/v3/project).

Подробно про Home — [отдельная страница](/v3/home): это главный экран, который ведёт человека через весь флоу.

## Чем V3 отличается от V1 и V2

| | V1 | V2 | V3 |
|---|---|---|---|
| Для кого | Обученная delivery-команда | Роли + автоматизация + admin | Любой человек без опыта |
| Как устроен | Модули и настройки | Workspace + агенты | Один guided-флоу |
| Роль AI | Помощник (генерация текстов) | Агенты и workflows | Исполнитель: AI делает, человек подтверждает |
| Порог входа | Дни обучения | Часы | Минуты |

<Note>V3 берёт решения из V2 (inline-создание объектов, AI-квалификация, Reply Queue-паттерн), но убирает всё ролевое разделение: один человек проходит весь путь от проекта до встречи.</Note>

## Открытые вопросы

| Вопрос | Почему важно |
|---|---|
| Права в команде | В V3 все участники равны. Достаточно ли этого, или нужен минимум «может отправлять инвойсы / не может»? |
| Автоотправка AI-ответов | В V3.0 AI никогда не отправляет сам. Когда доверие вырастет — включать ли auto-send для безопасных типов (out of office, unsubscribe)? |
| Переключение V1 ↔ V3 | Один проект — два интерфейса? Опытный пользователь захочет V1-контроль в отдельных местах |
