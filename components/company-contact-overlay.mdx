Полноэкранный оверлей, открывающийся при клике на любую Компанию или Контакт в любом месте приложения.

## Header

Быстрые действия доступны из любой табы:

| Кнопка | Действие |
|---|---|
| Add task | Создать задачу по компании/контакту |
| Book meeting | Забронировать встречу |
| Compose | Написать сообщение |

## Структура

**Слева** — основной контент выбранной табы (переписка, таблица задач и т.п.).

**Справа** — панель **Details** с полями Company и Contact. Если к компании привязано несколько контактов — переключение через навигацию `Contact 1/2 < >`.

## Контекстное открытие

Оверлей открывается в нужном состоянии в зависимости от того, откуда был вызван. Например:
- Клик на тред в **Inbox** → открывается таба Emails с уже открытым этим тредом
- Клик на компанию в **Leads** → открывается таба Overview

Внутри каждой табы можно идти глубже — например в Emails нажать на тред и откроется именно эта переписка.

## Табы

### Overview

Мини-дашборд по сущности — вся ключевая информация без переключения табов.

**Для Company:** базовая инфо (сайт, индустрия, размер), ключевые цифры (контактов, отправлено писем, встреч), статус последнего треда, дата последней активности, открытые задачи.

**Для Contact:** имя, должность, email, LinkedIn, телефон, привязанная компания, последнее письмо и когда, открытые задачи.

<Info>Финальное содержимое Overview в процессе проработки.</Info>

### Activity

Таблица с историей активности — кто добавил компанию/контакт, кто и когда написал письмо, какие данные были изменены.

Есть фильтрация по типу активности. Нужно добавить **Quick Filters** для быстрого переключения между типами: Email / LinkedIn / Call / Meeting / Task / Data change.

### Contacts *(только для Company)*

Таблица контактов, привязанных к этой компании.

**Add contact** открывает поп-ап с тремя способами добавления:

| Таба | Как работает |
|---|---|
| **LinkedIn** | Вставить ссылку на LinkedIn-профиль → данные парсятся через **Generect API** автоматически |
| **Email** | Ввести email → находим LinkedIn профиль через **Apollo** → данные парсятся через Generect API |
| **Manual** | Заполнить поля вручную |

Если профиль не найден или LinkedIn приватный — показывается ошибка.

### Emails

Таблица всех email-тредов с компанией. Тред принадлежит компании если в нём есть хотя бы один контакт из этой компании.

#### Просмотр треда

При клике на тред открывается страница переписки внутри оверлея. Сообщения отображаются в хронологическом порядке, длинные треды сворачиваются с кнопкой "Show N more messages".

#### Compose / Reply

В нижней части страницы треда — форма ответа:

| Элемент | Описание |
|---|---|
| From | Email-ящик отправителя — подставляется автоматически |
| To | Получатель. Поддерживает CC/BCC |
| Body | Текстовое поле. "Start typing or select a template" |

**Панель инструментов:**
- **[AI Editor](/components/ai-editor)** — цветная звёздочка, отдельный компонент (описан отдельно)
- **Quick AI actions** — быстрые AI-действия (дропдаун)
- **Форматирование** — Bold, Italic, списки, ссылки и др.
- **Attach** — прикрепить файл
- **Create task** — создать задачу; в поп-апе автоматически подставляется тип `Follow-up in thread` и данные компании и треда
- **Send** — отправить
- **Schedule** — запланировать отправку на определённое время

#### Reply / Reply to all

<Warning>
  Нужно добавить кнопки **Reply** и **Reply to all** под каждым сообщением в треде (как в Gmail). При клике — поле TO заполняется автоматически из этого письма. Сейчас TO заполняется вручную, что неудобно и создаёт риск ошибки.
</Warning>

### LinkedIn

Аналогично табе [Emails](#emails) — таблица LinkedIn-тредов с компанией и та же функциональность внутри треда. Отличие только в канале коммуникации.

### Calls

Все звонки с компанией или контактом.

### Tasks

Тот же функционал что и в [модуле Tasks](/features/tasks), но с автоматической фильтрацией по текущей компании или контакту.

### Meetings

Тот же функционал что и в [модуле Meetings](/features/meetings), но с автоматической фильтрацией по текущей компании или контакту.

### Notes

Таблица заметок с автоматической фильтрацией по текущей компании или контакту. Видны всей команде — добавлять и редактировать может любой участник проекта.

## Поля Details

### Company

| Поле | Описание |
|---|---|
| Name | Название компании |
| Website | Сайт |
| LinkedIn | Страница компании в LinkedIn |
| Industry | Индустрия |
| Location | Местоположение |
| Employees | Количество сотрудников |
| ESP | Email Service Provider — определяется автоматически |
| Lists | [Списки](/features/settings/lists), в которые добавлена компания. Кнопка `+` для добавления в список. |

### Contact

| Поле | Описание |
|---|---|
| Name | Имя контакта |
| Company | Привязанная компания |
| Job Title | Должность |
| Job started on | Дата начала работы на текущей должности — сигнал для outreach |
| LinkedIn | Профиль в LinkedIn |
| Email | Email-адрес |
| Phone | Телефон |
| ESP | Email Service Provider — определяется автоматически |
| Location | Местоположение |

## Запланировано

**Notes / Comments в треде** — таба рядом с Details внутри email-треда для комментариев конкретно к этой переписке. Отличается от общих Notes компании/контакта.
