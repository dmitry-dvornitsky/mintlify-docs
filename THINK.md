# ПОДУМАТЬ — идеи для будущего

Идеи по улучшению UX и фич, которые не срочны но стоит вернуться.

---

## Company & Contact Overlay — Activity tab

- **Группировка по дате** — Today / Yesterday / This week / Earlier. Плоская таблица теряет хронологический контекст.
- **Фильтр по пользователю** — кто из команды сделал действие. Нужно Account Strategist для контроля работы SDR.
- **Date range** — показать активность за 7 / 30 / 90 дней.
- **Timeline view** — альтернатива таблице. Активность по природе хронологична, таймлайн читается естественнее.

## Overlay / Emails — Reply UX

Добавить **Reply** и **Reply to all** под каждым сообщением в треде. При клике TO заполняется автоматически из этого письма. Сейчас TO вводится вручную — риск ошибки и лишний шаг для SDR.

## Company & Contact Overlay — Header

**Текущее:** Add task, Book meeting, Compose

**Идеи:**
- Заменить Compose на две отдельные кнопки: **Email** и **LinkedIn** (с иконками) — чётче показывает канал
- Добавить **Log call** — быстрый способ залогировать звонок без захода в таб Calls
- Добавить **⋯ More** для редких действий (добавить в список, удалить, merge дубликатов)

Итоговый хедер: `[Add task] [Book meeting] [Log call] [Email] [LinkedIn] [⋯]`

## Tasks — секции таблицы

**Текущее:** Overdue / Scheduled / Done

**Проблема:** "Scheduled" размытый — не видно что нужно сделать сегодня vs на следующей неделе.

**Идея:** разбить на **Overdue / Today / Upcoming / Done**. Секция Today даёт SDR чёткий приоритет на день без необходимости сканировать весь список Scheduled.

## Requests — упрощение

**Текущее:** Import Request = контейнер для группировки нескольких импортов по тематике (например, компании с одной конференции). Внутри реквеста — Companies + Imports табы.

**Проблема:** лишний уровень вложенности. В реальности никто не группирует импорты по тематике — это просто overhead при каждом импорте.

**Идея:** убрать концепцию Import Request, заменить на плоскую **историю импортов** с колонками: дата, тип, количество записей, статус, источник. Контекст источника можно передавать через тег/метку к каждому импорту, не через контейнер.

## Leads / Import Companies — Step 2

**Проблема:** Step 2 перегружен — на одном экране одновременно настройки импорта (Import Name, Intent, DNC, Skip), конфигурация обогащения (Revenue filter) и результаты валидации (Valid/Invalid). Непонятная последовательность.

**Идея разделить на отдельные шаги:**
1. Данные (CSV / LinkedIn URLs)
2. Настройки импорта (Name, Lead Type, DNC, Skip)
3. Валидация — только Valid/Invalid результаты с Export/View
4. (Опционально) Revenue enrichment как отдельный шаг или post-import действие

Step 3 удалён как неудачный.

## Insights

**Client/Products дублирует Product табы** — в Client (Business) есть раздел Products, и одновременно есть отдельные Product вкладки в сайдбаре. Непонятно что где искать.

**Storage как свалка** — файлы вперемешку (транскрипты, деки, ICP). ICP — стратегический документ который SDR и Content Writer читают постоянно, он теряется среди остального. Возможно ICP стоит вынести в отдельный структурированный раздел.

**Сайдбар растёт бесконтрольно** — если у клиента 3+ продукта, сайдбар становится длинным и неудобным.

**Add page → полный экран редактора** — выбивает из контекста модуля.

## Settings / Products

**Проблема:** таблица для обычно одного продукта — избыточный UX.

**Идея:** заменить таблицу на форму с полями прямо в разделе (Name, Description, Competitors). Внизу кнопка **"+ Add another product"** для редких случаев с несколькими продуктами. Продукт как сущность остаётся — просто UX становится проще для типичного случая.
