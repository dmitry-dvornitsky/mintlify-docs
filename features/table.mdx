Таблица — главный UI-компонент приложения. Используется практически везде: Projects, Leads, Campaigns, Reporting и других модулях.

## Структура

### Header

Слева:
- **Sort** — сортировка по колонкам, в том числе multi-sort (по нескольким колонкам)
- **Filters** — расширенная фильтрация. Когда фильтры применены показывает `Filtered by {Field}`, при нескольких — добавляет `+N`
- **Quick Filters** — быстрые предустановленные фильтры

Справа:
- **View** — управление отображением: видимость колонок, порядок колонок
- **Action** — действия над данными, набор кнопок зависит от таблицы (часто — Create)

### Колонки

По клику на заголовок колонки открывается меню:

| Действие | Описание |
|---|---|
| Sort Ascending | Сортировка по возрастанию |
| Sort Descending | Сортировка по убыванию |
| Move Left | Переместить колонку влево |
| Move Right | Переместить колонку вправо |
| Hide from view | Скрыть колонку |

Также поддерживается изменение ширины колонки перетаскиванием границы заголовка.

### Body

Первая колонка — чекбоксы для выбора строк. При клике на главный чекбокс в хедере появляется меню:

| Опция | Статус |
|---|---|
| Select page | ✅ Есть |
| Select all table | ✅ Есть |
| Select range | 🔲 Нужно добавить |

При выборе строк внизу экрана появляется **Bulk Action Bar** — панель с действиями для работы с выбранными записями. Набор действий зависит от таблицы.

В некоторых таблицах при наведении на строку появляются **Quick Actions** прямо в строке.

### Footer

Слева:
- Общее количество записей в таблице

Справа:
- **Rows per page** — селектор количества строк на странице
- **Pagination** — навигация по страницам

## Filters

Расширенная фильтрация. Когда фильтры применены — хедер показывает `Filtered by {Field}`, при нескольких полях добавляет `+N`.

### Структура фильтра

Каждый фильтр состоит из трёх частей:

```
[Property] → [Condition] → [Value]
```

**Property** — поле по которому фильтруем. Можно фильтровать по полям самой сущности и по полям связанных сущностей (вложенная фильтрация). Навигация по связанным сущностям иерархическая с поиском внутри.

Пример: `User → Full name` — фильтр по имени пользователя связанного с записью.

### Типы полей и их Conditions

Набор conditions зависит от типа поля.

**String**

| Condition | Описание |
|---|---|
| Contains | Содержит |
| Not Contains | Не содержит |
| Starts With | Начинается с |
| Ends With | Заканчивается на |
| Equals | Равно |
| Not Equals | Не равно |
| Is Empty | Пусто |
| Is Not Empty | Не пусто |

**Date**

| Condition | Описание |
|---|---|
| Equals | Равно дате |
| Not Equals | Не равно дате |
| Before | До даты |
| After | После даты |
| Is Empty | Пусто |
| Is Not Empty | Не пусто |

**Number, Array, Boolean** — у каждого свои conditions (нужно уточнить и добавить).

## Quick Filters

Предопределённый набор наиболее востребованных фильтров для конкретной таблицы. Проще и быстрее чем Filters — не нужно выбирать property и condition, только значения.

### Как работает

Открывается дропдаун со списком предустановленных фильтров. Каждый фильтр — отдельный пункт с иконкой. При наведении открывается подменю с чекбоксами для выбора значений (мульти-селект).

Пример Quick Filters из Inbox:

| Фильтр | Значения |
|---|---|
| Reply Type | — |
| Channel | Email, LinkedIn |
| Read Status | — |
| Campaign Name | — |
| Contacts in Campaign | — |
| Last Contact Date | — |

Quick Filters специфичны для каждой таблицы — набор разный в зависимости от модуля.
