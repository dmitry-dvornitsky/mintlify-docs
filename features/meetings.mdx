Meetings — таблица всех встреч, назначенных клиенту. Это главный KPI delivery команды.

## Требования

Перед бронированием встречи необходимо подключить минимум один календарь в [Settings → Calendars](/features/settings/calendars).

Поддерживаемые интеграции:
- **Google Calendar**
- **Microsoft Calendar**
- **Calendly** — интеграция доступна, но не используется

## Бронирование встречи

Кнопка **Book Meeting** открывает поп-ап с полями:

| Поле | Описание |
|---|---|
| Calendar | Выбор подключённого календаря |
| Time slot | Выбор времени встречи |
| Company | Компания, с которой бронируется встреча — подтягиваются только компании из [Leads → Companies](/features/leads) |
| Contact | Контакт из выбранной компании |
| Channel | Канал через который договорились о встрече: **Email**, **LinkedIn**, **Call** |

## Статусы встреч

Каждая встреча имеет статус:

| Статус | Описание |
|---|---|
| Scheduled | Встреча запланирована |
| Completed | Встреча состоялась |
| Cancelled | Отменена |
| No Show | Контакт не появился |

В [KPI](/features/settings/kpi) засчитываются только встречи со статусом **Completed**.

## Еженедельный review

Каждую неделю проходит звонок с клиентом, на котором команда проставляет актуальные статусы и добавляет комментарии к встречам.

## Slack-уведомления

После создания встречи автоматически отправляется алерт в Slack-канал.

Для этого нужно подключить канал в [Settings → Slack](/features/settings/slack).
