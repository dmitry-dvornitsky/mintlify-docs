Leads — база компаний и контактов для outreach. Разделён на две табы: **Companies** и **Contacts**.

Тесно связан с модулем [Requests](/features/requests) — история всех импортов и поисков хранится там.

## Companies

Таблица компаний проекта.

**Actions:**
- **Import companies** — импорт компаний из CSV или списка LinkedIn URL
- **Find companies** — поиск компаний по ICP-критериям

### Find Companies

Форма поиска компаний по параметрам. Результаты идут в Leads → Companies и сохраняются в Requests → Find Companies.

Поля с `*` обязательны.

#### Company details

| Поле | Описание |
|---|---|
| Locations * | Страны и регионы. Мульти-селект с поиском, иерархический (страна → штат/город). Можно выбрать "All" для всей страны или углубиться. |
| Industry * | Индустрии. Мульти-селект с поиском и иерархией. |
| Exclude Industries | Индустрии которые нужно исключить. |
| Type * | Тип компании: Public Company, Privately Held, Non Profit, Partnership, Government Agency, Educational, Self Employed, Self Owned. |
| Company size * | Размер по количеству сотрудников (например, 11–50). |
| Keywords | Ключевые слова для поиска — вводятся тегами. |
| Exclude keywords | Ключевые слова для исключения. |

#### Department headcount

| Поле | Описание |
|---|---|
| Department | Конкретный департамент внутри компании |
| Headcount min / max | Размер департамента |

#### Revenue range

| Поле | Описание |
|---|---|
| LinkedIn revenue min / max ($M) | Диапазон выручки по данным LinkedIn |
| Revenue min / max ($M) | Диапазон выручки по другим источникам |

#### Search options

| Опция | Описание |
|---|---|
| Collect revenue | Подтянуть данные о выручке для найденных компаний |
| Skip range revenue | Пропустить компании вне заданного диапазона выручки |
| Small companies first | Показывать маленькие компании первыми (проще выйти на ЛПР) |

#### Step 2 — Preview & Name

Сводка всех выбранных критериев для проверки перед запуском поиска. Поле **Name*** — название этого поиска, сохраняется в [Requests → Find Companies](/features/requests) как именованный Request.

#### Как работает пагинация

LinkedIn отдаёт ограниченное количество результатов на один запрос (несколько тысяч). Чтобы выгрузить больше компаний из большого региона (например, US), нужно разбивать поиск на части — например, по штатам.

Для этого используется **Request** как контейнер ICP-сегмента. Внутри Request можно запускать несколько **Searches** последовательно. Каждый новый Search видит что уже было найдено (Skip Already Searched Companies) и не дублирует результаты.

**Страница Request (Requests → Find Companies → {Name}):**

Статистика вверху:
- Number of found companies
- Number of processed companies
- Companies on LinkedIn
- Average company price

Табы: **Companies** (все найденные компании) | **Searches** (история запусков)

Колонки таблицы Searches: Actions, Status, Name, Requested Amount, Found Companies, Processed Companies

**Запуск нового Search** (кнопка "New search"):

| Поле | Описание |
|---|---|
| Companies * | Сколько компаний найти в этом запуске |
| Apply Companies DNC List | Пропустить компании из DNC (по умолчанию включено) |
| Skip Already Searched Companies | Не повторять уже найденные (по умолчанию включено) |
| Run Company AI Analysis after search completion | Автоматически запустить AI-анализ после завершения |

### Import Companies

Мульти-степ флоу из 3 шагов.

---

#### Step 1 — Данные

Два способа загрузки:
- **CSV** — загрузить файл
- **LinkedIn URLs** — вставить список ссылок на компании

---

#### Step 2 — Validation Results

<Warning>Этот шаг перегружен — планируется переработка UX. Пока документируем как есть.</Warning>

Экран совмещает настройки импорта и результаты валидации.

**Настройки:**

| Поле | Описание |
|---|---|
| Import Name | Название этого импорта для истории в Requests |
| Intent | **Warm** или **Cold** — знает ли лид о клиенте. Warm = есть связь (например, бывшие клиенты), Cold = нет. Переименовать в **Lead Type** или **Audience**. |
| Apply Companies DNC List | Пропустить компании из Do Not Contact списка |
| Skip Already Searched Companies | Пропустить компании которые уже есть в системе |

**Revenue filter** — опциональный фильтр по выручке:

| Поле | Описание |
|---|---|
| Collect revenue | Подтянуть данные о выручке для каждой компании |
| Revenue min / max ($M) | Диапазон выручки — компании вне диапазона помечаются |
| Skip range revenue | Пропустить компании вне диапазона |

**Результаты валидации:**

Два таба — **Valid** и **Invalid** — показывают какие записи прошли проверку. Кнопки View и Export для просмотра и скачивания.

## Contacts

Таблица контактов проекта.

**Actions:**
- **Import contacts** — импорт контактов
- **Find contacts** — поиск контактов по комбинации критериев компании и контакта

### Find Contacts in Companies

Поиск контактов внутри уже найденных компаний.

**Флоу:**
1. Leads → Companies — выбрать компании чекбоксами
2. В Bulk Action Bar нажать **Find contacts**
3. Открывается страница с критериями поиска контактов
4. Создаётся Request в Requests → Find Contacts in Companies

**Критерии поиска (New Contact Search):**

| Поле | Описание |
|---|---|
| Locations | Локация контактов (опционально) |
| Audiences — First Priority * | Основная целевая аудитория из [Settings → Audiences](/features/settings/audiences) |
| Audiences — Secondary Priority | Запасная аудитория (если контакты первой не найдены) |
| Max Contacts Per Company * | Максимум контактов с одной компании (например, 3) |
| Photo optional | Включать контакты без фото в LinkedIn |
| Job dates optional | Включать контакты без дат трудоустройства |
| Skip C-level on large headcount companies | Пропускать C-level в крупных компаниях (до них сложнее добраться) |
| Years in current company | Диапазон лет в текущей должности (слайдер: <1, 1-2, 3-5, 6-10, 10+) |
| Apply Contacts DNC list | Пропустить контакты из DNC (по умолчанию включено) |

**Страница Request (Requests → Find Contacts in Companies → {Name}):**

Статистика: Number of found contacts, Companies with found contacts

Табы: **Contacts** | **Companies** | **Searches**

Колонки таблицы Searches: Actions, Status, Name, Found Contacts, Processed Companies

<Warning>Флоу требует переработки — см. [redesigns/leads-find-contacts.md](/redesigns/leads-find-contacts).</Warning>

### Find Contacts

Поиск контактов напрямую — без предварительного выбора компаний. Вход: Leads → Contacts → **Find contacts**.

Форма разбита на две колонки: **Company details** (критерии компаний, в которых искать) и **Contact details** (критерии самих контактов). Результаты идут в Leads → Contacts и сохраняются в Requests → Find Contacts.

#### Company details

| Поле | Описание |
|---|---|
| Locations | Страны/регионы компании |
| Industry | Индустрии |
| Exclude Industries | Индустрии для исключения |
| Type | Тип компании (Public, Private, Non-profit и др.) |
| Company size | Размер по количеству сотрудников |
| Revenue min / max ($M) | Диапазон выручки |
| Collect revenue | Подтянуть данные о выручке |
| Skip range revenue | Пропустить компании вне диапазона |
| Small companies first | Показывать маленькие компании первыми |

#### Contact details

| Поле | Описание |
|---|---|
| Audiences (Job Titles) * | Целевая аудитория из [Settings → Audiences](/features/settings/audiences). Одиночный выбор |
| Locations | Локация контактов (отдельно от локации компании) |
| Max contacts per company | Максимум контактов с одной компании (по умолчанию 1) |
| Years in current company | Диапазон лет в текущей должности (слайдер: <1, 1-2, 3-5, 6-10, 10+) |
| Photo optional | Включать контакты без фото в LinkedIn |
| Job dates optional | Включать контакты без дат трудоустройства |
| Recently posted on LinkedIn | Искать только тех кто недавно был активен в LinkedIn |

<Note>
В отличие от **Find Contacts in Companies**, этот флоу не привязан к конкретным компаниям из Leads — компании находятся по критериям одновременно с поиском контактов.
Поле Audiences здесь одиночный выбор, без First/Secondary Priority.
</Note>
