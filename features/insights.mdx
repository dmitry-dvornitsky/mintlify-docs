Insights — база знаний по проекту. Хранит всю информацию о клиенте, его бизнесе и продуктах. Каждая страница выглядит как Notion-документ.

## Структура сайдбара

### 1. Storage

Файловое хранилище материалов от клиента:
- Транскрипты звонков
- Sales-деки
- ICP (Ideal Customer Profile)
- Другие файлы клиента

### 2. Client

Страницы с информацией о бизнесе клиента. Содержит список страниц (Title / Updated at / Updated By / Status):

| Страница | Содержание |
|---|---|
| Client Overview | Общая информация о компании |
| Client Expectations | Ожидания клиента от сотрудничества |
| Client Acquisition Channels | Текущие каналы привлечения клиентов |
| Client Products | Продуктовая линейка клиента |

### 3. Product (× N)

Отдельная вкладка для каждого продукта клиента — создаётся автоматически из [Settings → Products](/features/settings/products). Вкладки называются по имени продукта (например, "Business Loan").

Каждая продуктовая вкладка содержит разделы:

| Раздел | Содержание |
|---|---|
| Overview | Описание продукта |
| Pricing | Ценообразование |
| Solution | Что решает продукт |
| Competitor Comparison | Сравнение с конкурентами |
| Objections | Типичные возражения и ответы на них |
| **Strategies** | AI-сгенерированные стратегии для outreach — см. ниже |

## Strategies

Strategies — раздел внутри каждого Product. Список страниц (Title / Updated at / Updated By / Status) с кнопкой **Add page**.

Каждая Strategy страница генерируется AI на основе информации о продукте и данных из других разделов Insights. Контент представляет собой структурированный анализ: какие потребности рынка решает продукт и кому писать.

### Структура страницы

**Combined industry context (used for validation)** — сводное описание целевой аудитории, агрегированное из других страниц (Industry Context, ICP). Используется AI для квалификации лидов.

**Validated needs, mapped to [Product] solutions and outreach roles** — пронумерованный список потребностей. Каждый пункт:

```
[Название потребности]

Need:     Что болит у клиента — конкретная бизнес-проблема
Solution: Как именно продукт это решает (со ссылками на product pages)
Roles:    Кому писать — конкретные должности (COO, VP of Operations, CTO...)
```

Примеры страниц: "Needs Solutions And Roles", "Seasonal And Timing Insights", "Industry Context And Trends", "Industry Focus And Market".

### Связь с Campaigns

Strategy выбирается на Step 1 создания кампании. AI использует её как контекст при генерации текстов последовательности — понимает какие потребности закрывает продукт и каким ролям пишет.

## Создание страниц

Внутри каждого раздела — таблица страниц и кнопка **Add page**.

При нажатии **Add page** открывается редактор:
- **Слева** — чат с AI: можно попросить сгенерировать или улучшить контент
- **Справа** — текстовый редактор в стиле Notion

Можно писать вручную, через AI или комбинировать.
