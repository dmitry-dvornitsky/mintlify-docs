## General

Основная информация о проекте и клиенте.

### Информация о компании

| Поле | Описание | Статус |
|---|---|---|
| Name | Название клиента | ✅ |
| Slug | URL-friendly идентификатор проекта, автогенерируется из Name | ✅ |
| Size | Размер компании клиента | ✅ |
| Industry | Индустрия | ✅ |
| Country | Страна | ✅ |
| State | Штат / регион | ✅ |
| Timezone | Часовой пояс клиента — важно для таймингов outreach | 🔲 Нужно добавить |
| Storage Link | Ссылка на Google Drive по проекту | ✅ |
| Website | Сайт клиента | ✅ |
| Contact Person | Главный стейкхолдер на стороне клиента | 🔲 Нужно добавить |

### Контракт

| Поле | Описание |
|---|---|
| Customer Since / To | Период работы с клиентом |
| Contract Pilot | Длительность пилотного периода в месяцах (например, 3). В это время команда нарабатывает рабочие стратегии для проекта. |
| Contract Recurring | Рекаррин-период в месяцах (1, 3, 6 или любое другое число) |
| Contract Specifics | Детали контракта в свободной форме |

## KPI

Плановые показатели по встречам на каждый месяц проекта.

### Добавление KPI

Кнопка **Add KPI** открывает поп-ап с полями:
- **Month** — месяц
- **Year** — год
- **Goal** — целевое количество встреч

### Таблица

| Колонка | Описание |
|---|---|
| Month | Месяц |
| Year | Год |
| Goal | Плановое количество встреч |
| Result | Фактическое количество встреч — подтягивается автоматически из модуля Meetings |
| % | Процент выполнения KPI (Result / Goal) — вычисляемая колонка |
| Updated by | Кто последний редактировал Goal |

Удаление записей недоступно намеренно — для сохранения истории.

### Действия по строке

В конце каждой строки — меню с тремя действиями:

| Действие | Описание | Доступ |
|---|---|---|
| Edit | Редактировать Goal | Account Director, Account Strategist, Менеджеры с Admin доступом |
| KPI History | Поп-ап с историей изменений — кто и что менял | Все |
| Meetings History | История встреч за месяц, к которому относится KPI | Все |

## Billing

<Info>Раздел описан поверхностно, требует детального описания позже.</Info>

Раздел содержит финансовую информацию по клиенту, организован в 4 таба.

### Billing Information

Контактные и юридические данные для выставления инвойсов:

| Поле | Описание |
|---|---|
| Legal name | Юридическое название компании |
| Location | Адрес |
| Email | Email для инвойсов |
| First name / Last name | Контактное лицо |
| NET | Срок оплаты инвойса в днях (например, NET 30 = 30 дней) |

### Services

Подписки и one-time сервисы, предоставляемые клиенту.

### Invoices

Таблица инвойсов для клиента.

### Churn & Downgrade

История чарнов и даунгрейдов клиента. Записи добавляются вручную для трекинга сумм и дат.

## Products

<Warning>Спорная фича — рассмотреть переработку. Описание проблемы ниже.</Warning>

Таблица продуктов и сервисов клиента. Продукты связываются с документами и стратегиями в модуле Insights.

### Добавление продукта

Кнопка **Add product** открывает поп-ап с полями:
- **Name** — название продукта
- **Description** — описание
- **Competitor websites** — мульти-инпут для ссылок на сайты конкурентов

### Таблица

| Колонка | Описание |
|---|---|
| Name | Название продукта |
| Description | Описание |
| Competitors | Ссылки на сайты конкурентов |
| Updated at | Дата последнего обновления |

### Проблема

На практике у большинства клиентов **один продукт**. Таблица для одной строки — избыточный UX. При этом продукт нужен как сущность для привязки документов в Insights.
