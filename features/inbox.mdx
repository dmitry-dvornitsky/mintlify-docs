Inbox — единое рабочее пространство для переписки с лидами через Email и LinkedIn. Оба канала приведены к общему формату данных и отображаются вместе без разделения.

## Thread

Главная сущность модуля. Каждый Thread — это переписка с одним контактом.

| Поле | Описание |
|---|---|
| Status | **Open** / **Closed**. При закрытии нужно выбрать причину. Closed — лид не заинтересован или не отвечает, писать в тред больше нет смысла. |
| Type | Характер ответа лида: Interested, Not Interested, Redirect, Unsubscribe и др. |
| Labels | Метки из [Settings → Labels](/features/settings/labels). Назначаются на тред. |
| From | Email-ящик или LinkedIn-аккаунт, с которого ведётся переписка в этом треде. |

## Сайдбар

Основные разделы для SDR:

| Раздел | Описание |
|---|---|
| **Inbox** | Актуальные треды — только важное, без мусора |
| **Scheduled** | Запланированные отправки |
| **Drafts** | Черновики сообщений |
| **Sent** | Отправленные сообщения |
| **All mails** | Все сообщения вперемешку |

Служебные разделы (внизу сайдбара, в основном для EDS):

| Раздел | Описание |
|---|---|
| **Bounces** | Email-баунсы |
| **Archive** | Архив — тред полностью скрывается, не появляется даже в All mails |
| **Warmup** | Warmup-сообщения |

## Archive vs Closed

| | Archive | Closed (статус треда) |
|---|---|---|
| Где виден | Нигде — скрыт полностью | В All mails |
| Когда использовать | Убрать мусор и нерелевантные треды | Лид не заинтересован или не отвечает |
| История сохраняется | Да, но скрыта | Да, доступна |
