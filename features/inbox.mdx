Inbox — главный инструмент SDR для работы с перепиской. Email и LinkedIn приведены к единому формату: все треды отображаются вместе, без разделения по каналу.

## Навигация

Левый сайдбар разделён на два блока.

**Основные разделы:**

| Раздел | Что показывает |
|---|---|
| **Inbox** | Активные открытые треды — то что требует внимания |
| **Scheduled** | Сообщения, запланированные к отправке в будущем |
| **Drafts** | Сохранённые черновики |
| **Sent** | Отправленные сообщения |
| **All mails** | Все треды без исключений — включая закрытые |

**Служебные разделы** (внизу, в основном для EDS):

| Раздел | Что показывает |
|---|---|
| **Bounces** | Email-баунсы — письма которые не доставились |
| **Archive** | Заархивированные треды — скрыты полностью, не появляются даже в All mails |

## Таблица тредов

Каждый раздел отображается как таблица тредов.

**Actions:** Sort, Filter, **Quick Filters**

### Колонки

| Колонка | Описание |
|---|---|
| **Contact** | Имя контакта. Клик открывает [Company & Contact Overlay](/components/company-contact-overlay) |
| **Company** | Компания контакта |
| **Channel** | Иконка канала — Email или LinkedIn |
| **Subject / Preview** | Тема письма или превью последнего сообщения |
| **Type** | Тип ответа лида — классифицируется AI автоматически. [Список типов ниже](#тип-ответа) |
| **Labels** | Метки назначенные на тред из [Settings → Labels](/features/settings/labels) |
| **From** | Email-ящик или LinkedIn аккаунт через который ведётся переписка |
| **Date** | Время последнего сообщения в треде |

### Quick Filters

Быстрая фильтрация тредов по наиболее частым критериям:

- **Channel** — Email / LinkedIn
- **Type** — по типу ответа (Interested, Not Interested и др.)
- **Labels** — по конкретной метке
- **Status** — Open / Closed

## Тред

Главная сущность Inbox. Каждый тред — это переписка с одним контактом. Объединяет все сообщения с этим контактом в хронологическом порядке.

Клик на тред в таблице открывает его содержимое. Длинные треды сворачиваются с кнопкой **Show N more messages**.

Параллельно справа всегда видна панель **Details** с данными компании и контакта — не нужно переключаться в другой раздел чтобы вспомнить кто это.

### Статус

Каждый тред находится в одном из двух статусов:

| Статус | Описание |
|---|---|
| **Open** | Тред активен — требует внимания или в процессе |
| **Closed** | Тред закрыт. При закрытии нужно выбрать причину |

Треды со статусом **Closed** не отображаются в разделе **Inbox**, но доступны в **All mails**.

Закрыть тред = сигнал что этот лид больше не в работе. Причина закрытия — дополнительный контекст для команды и аналитики.

### Тип ответа

AI автоматически классифицирует каждый входящий ответ и выставляет тип на тред. SDR может изменить тип вручную.

| Тип | Описание |
|---|---|
| **Appointment** | Контакт согласился на встречу |
| **Interested** | Проявил интерес, встреча ещё не назначена |
| **Talking** | Диалог продолжается |
| **Contact later** | Просит связаться позже |
| **Forwarded** | Переадресовал на другого человека |
| **Not interested** | Явный отказ |
| **Unsubscribe** | Просит отписать от рассылки |
| **DNC** | Do Not Contact — жёсткий отказ от любой коммуникации |
| **Ghosted** | Ответил ранее, затем перестал отвечать |
| **Automation** | Автоответ (out of office и аналогичные) |
| **Unknown** | AI не смог классифицировать ответ |

Те же типы отображаются в аналитике кампаний — [Reply Types в Campaign → Email](/features/campaigns#reply-types).

### Метки (Labels)

На тред можно назначить одну или несколько меток из [Settings → Labels](/features/settings/labels). Метки помогают классифицировать треды по внутренней логике команды — например «Нужно позвонить», «Передать в Sales», «VIP».

## Ответ на сообщение

Форма ответа расположена внизу открытого треда.

| Поле | Описание |
|---|---|
| **From** | Email-ящик или LinkedIn аккаунт отправителя — подставляется автоматически из того же канала что и тред |
| **To** | Получатель |
| **Body** | Текст сообщения. Подсказка «Start typing or select a template» — доступны шаблоны из [Settings → Message Templates](/features/settings/message-templates) |

**Панель инструментов:**

| Элемент | Описание |
|---|---|
| **✦ AI Editor** | AI-редактор для генерации текста с учётом контекста треда, компании и продукта. [Подробнее →](/components/ai-editor) |
| **Quick AI actions** | Быстрые AI-действия без открытия полного редактора |
| **Форматирование** | Bold, Italic, списки, ссылки |
| **Attach** | Прикрепить файл |
| **Create task** | Создать задачу по треду — автоматически подставляется тип [Follow-up in thread](/features/tasks#follow-up-in-thread) и привязка к контакту |
| **Send** | Отправить сообщение |
| **Schedule** | Запланировать отправку на конкретное время |

<Note>
Reply и Reply to all под каждым сообщением в треде — в разработке. Сейчас поле TO заполняется вручную.
</Note>

## Archive vs Closed

Два механизма убрать тред «с глаз» — с разным смыслом:

| | Archive | Closed |
|---|---|---|
| **Где виден** | Нигде — скрыт полностью, даже в All mails | Доступен в All mails |
| **Когда использовать** | Мусор: нерелевантные треды, ошибочные переписки, служебные письма | Лид не заинтересован или переписка завершена |
| **Требует причины** | Нет | Да |
| **История сохраняется** | Да, но скрыта | Да, доступна |

## Scheduled

Сообщения, запланированные к отправке в будущем. SDR может запланировать ответ через кнопку **Schedule** в форме ответа — сообщение уйдёт в указанное время автоматически.

Из раздела Scheduled можно отменить запланированную отправку или изменить время.

---

## Drafts

Черновики — сообщения которые начали писать, но не отправили. Сохраняются автоматически.

---

## Sent

Все отправленные сообщения — как из Inbox, так и из Overlay. Позволяет проверить что и когда было отправлено.

---

## All mails

Полный список тредов без фильтрации: открытые, закрытые, все каналы, все ящики. Нужен когда требуется найти конкретный тред который не попадает в основной Inbox.

---

## Bounces

Email-баунс — письмо не доставилось. Два типа:

| Тип | Описание |
|---|---|
| **Hard bounce** | Email не существует или домен недоступен. Контакт помечается автоматически |
| **Soft bounce** | Временная проблема — переполненный ящик, временная недоступность сервера |

Раздел Bounces используется EDS для мониторинга состояния ящиков и репутации домена.
