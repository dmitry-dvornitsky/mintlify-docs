Таблица продуктов и сервисов клиента. Продукты связываются с документами и стратегиями в модуле [Insights](/features/insights).

## Добавление продукта

Кнопка **Add product** открывает поп-ап с полями:
- **Name** — название продукта
- **Description** — описание
- **Competitor websites** — мульти-инпут для ссылок на сайты конкурентов

## Таблица

| Колонка | Описание |
|---|---|
| Name | Название продукта |
| Description | Описание |
| Competitors | Ссылки на сайты конкурентов |
| Updated at | Дата последнего обновления |

## Известная проблема

На практике у большинства клиентов **один продукт**. Таблица для одной строки — избыточный UX. При этом продукт нужен как сущность для привязки документов в Insights.

Идея по переработке — в [THINK.md](/THINK#settings--products).
