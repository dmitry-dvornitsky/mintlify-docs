Requests — журнал всех операций с лидами: импорты, поиски, enrichment. Организован в табы по типу операции.

## Табы

| Таба | Описание |
|---|---|
| **Find Companies** | История поисков компаний |
| **Find Contacts** | История поисков контактов |
| **Find Contacts in Companies** | История поисков контактов внутри компаний |
| **Import Companies** | История импортов компаний |
| **Import Contacts** | История импортов контактов |
| **Company AI Analysis** | История AI-анализов сайтов компаний |
| **LinkedIn Activity** | История LinkedIn активности |

## Текущая структура Import Request

Каждый импорт создаёт **Import Request** — контейнер который может содержать несколько импортов внутри.

Внутри Import Request:

**Статистика:**
- Number of imported companies
- Average company price — стоимость обогащения на компанию (Generect и другие инструменты)

**Табы:**
- **Companies** — все компании из всех импортов в этом реквесте (Company Name, Domain, LinkedIn URL, Industry)
- **Imports** — список отдельных импортов внутри реквеста

**Колонки таблицы Import Companies:**

| Колонка | Описание |
|---|---|
| Request Name | Название реквеста |
| Last Import Status | Статус последнего импорта |
| Companies imported | Количество импортированных компаний |
| Created at | Дата создания |

## Проблема

Текущая концепция: один Import Request = несколько импортов одной тематики (например, все компании с одной конференции). Идея — хранить контекст источника лидов.

На практике это лишний шаг который никому не нужен. Планируется упростить до плоской истории импортов.
